# v33 경로별 기존 연차대장 입력 지원

## 1. 요청 반영 방향

홈플러스와 전자랜드의 기존 연차대장 표 구조가 서로 달라도, 각 경로에서 현재 사용하는 파일을 그대로 기준자료로 등록할 수 있도록 입력 파서를 분리했습니다.

- 입력: 경로별 기존 파일 형식 유지
- 저장: 시스템 표준 연차 누적 원장으로 통합
- 이후 누적: 기존 월별 승인·반려 자료 누적 방식 유지

기존 파일을 수기로 수정해 다시 등록할 수 있으며, 같은 경로의 기준대장은 마지막 업로드 파일 기준으로 교체됩니다.

## 2. 홈플러스 기존 대장 인식

다음 형태의 단일 시트를 자동으로 찾습니다.

- 제니엘사번 또는 사번
- 사원명
- 구분 1 / 구분 2
- 지역 / 매니저 / 포탈사번
- 제니엘 입사일 / 연차기준일 / 퇴사(예정)일
- 기준연도 발생 / 사용가능 / 사용 / 잔여
- 연차발생(예정)일 / 적용 기간 / 비고

### 반영 기준

- 기준일 이전 또는 당일 퇴사: 제외
- 홈플러스→전자랜드: 제외
- 홈플러스→코스트코: 제외
- 전자랜드→홈플러스: 포함
- 육아휴직: 기준대장에는 포함
- 재직·촉탁직: 포함
- 1년 미만 여부: 구분 2와 근속기간 문구로 판정

### 수기 입력 처리

파일을 수정한 뒤 다시 등록하면 수식 식 자체를 해석하지 않고, XLSX에 저장된 현재 셀 값을 읽습니다.

- 사용가능과 잔여가 있으면 누적 사용 = 사용가능 - 잔여
- 잔여가 없으면 사용가능 - 사용으로 계산
- 적용 기간과 연차발생일도 현재 저장값을 반영

## 3. 전자랜드 기존 대장 인식

다음 두 종류를 지원합니다.

1. `1년 이상자`, `1년 미만자`처럼 분리된 기존 연차대장
2. `연차 누적 현황` 표준 시트

인식 필드는 다음과 같습니다.

- 구분
- 지역장 / 매니저 / 지역
- 점포코드 / 점포
- 스핀사번 / 사번 / 이름
- 입사일 / 연차 기준 입사일
- 사용기간 시작 / 종료
- 발생 / 누적 사용 / 잔여
- 1차·2차 촉진 / 소진일 / 비고

전자랜드도 기존 양식에서 값을 수기로 변경한 뒤 다시 등록할 수 있습니다.

## 4. 표준 누적 관리 유지

경로별 입력 파일의 모양은 달라도 저장 후에는 기존 표준 누적 원장 구조로 관리합니다.

- 사번별 기준 발생·사용·잔여
- 1년 이상 / 1년 미만 구분
- 사용기간과 촉진일
- 월별 승인 사용분 누적
- 같은 월 월별 파일 재등록 시 기존 월 자료 교체
- 경로별 자료 분리

따라서 홈플러스 양식을 전자랜드 모양으로 바꾸거나, 전자랜드 양식을 홈플러스 모양으로 바꿀 필요가 없습니다.

## 5. 실제 홈플러스 파일 검증

첨부된 `홈플러스 판매상담사 연차관리 _ 5월 마감(2).xlsx`를 기준일 2026-05-31로 읽어 검증했습니다.

- 인식 형식: 홈플러스 기존 단일 연차대장
- 반영 인원: 92명
- 제외 인원: 69명
- 1년 미만: 14명
- 기준 잔여 합계: 1,162일

퇴사자와 홈플러스에서 타 경로로 이동한 인원은 제외되고, 전자랜드에서 홈플러스로 이동한 인원과 육아휴직자는 포함되는 것을 확인했습니다.

## 6. 추가 보완

- 이름 뒤 숫자가 붙은 동명이인 표기(예: 김현수2)를 정상 사원명으로 인정
- 업로드 미리보기에서 자동 인식 형식, 읽은 시트, 반영/제외 인원, 1년 이상/미만, 잔여 합계 표시
- 경로 선택 시 해당 경로의 지원 양식 안내 표시
- 원본 파일은 월별 파일 보관함에도 함께 저장

## 7. 데이터 보호

- 신규 D1 마이그레이션 없음
- 기존 연차 누적 및 월별 사용내역 삭제 없음
- 기존 월 마감·근태·증빙·인력변동 데이터 삭제 없음
- 대체휴무·보상휴가 계산 변경 없음
- 기준대장 재등록은 해당 경로 기준자료만 교체

- `public/assets/app.js` JavaScript 구문 검사 통과
- `public/assets/final-template.js` JavaScript 구문 검사 통과

- 전자랜드 표준 원장 샘플 파싱 검증 통과
- 앱 표시 버전 및 정적 파일 캐시를 v33으로 갱신
