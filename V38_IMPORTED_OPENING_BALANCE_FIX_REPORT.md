# V38 이전 최종본 이월 잔여 인식 수정

## 확인한 원인

전자랜드 2026년 5월 최종본의 `5월 상담사근태` 시트에서 김형삼(E247645)은 다음처럼 기록되어 있습니다.

- 기본 휴무 기준: 10일
- 실제 휴무: 11일
- 수기 입력된 `대체휴무 이월개수`: 2일
- 5월 발생 대체휴무: 1일

따라서 5월 휴무 초과 1일은 5월 발생분 1일로 대체되고, 기존 이월분 2일은 6월에 그대로 남아야 합니다.

기존 코드는 상담사근태의 날짜 셀에서 `대체휴무`, `보상휴가` 사용표시만 읽었고, 요약부의 `대체휴무 이월개수` 수기 입력값은 읽지 않았습니다. 또한 해당 머리글은 윗줄 `대체휴무 이월개수`, 아랫줄 `입력요망`으로 나뉘어 있어 한 행 머리글 탐색으로는 찾을 수 없었습니다.

## 수정 내용

- 상담사근태 머리글을 최대 3행 합쳐서 탐색
- 아래 수기 열을 기초 이월 잔여로 인식
  - 대체휴무 이월개수 / 이월잔여 / 기초잔여
  - 보상휴가 이월개수 / 이월잔여 / 기초잔여
- 사번별 기초 이월 잔여를 D1 월 원본에 저장
- 최초 기준 월보다 앞선 부여분은 기초잔여와 중복 합산하지 않음
- 기준 월 이후 발생한 신규 대체휴무·보상휴가는 기존처럼 별도 추가
- 같은 경로·같은 월 최종본을 다시 반영하면 기존 기초잔여를 교체
- 정상 월 마감을 같은 월에 다시 저장해도 최초 최종본의 기초잔여는 보존
- 이월 잔여에 만료일 정보가 없는 과거 파일은 소진될 때까지 유지
- 파일 반영 미리보기와 완료 알림에 `이월 대체`, `이월 보상` 합계를 표시

## 실제 파일 검증

- 전자랜드 5월 최종본에서 이월 대체휴무 열 인식: 정상
- 김형삼(E247645) 이월 대체휴무: 2일 인식
- 전자랜드 전체 양수 이월 인원: 104명
- 전자랜드 이월 대체휴무 합계: 165일
- 홈플러스 5월 최종본도 동일한 2단 머리글 구조 인식

김형삼 계산:

- 기초 이월 2일 + 5월 신규 발생 1일 - 5월 휴무 초과 대체 1일 = 6월 이월 2일

## 배포 후 필요한 작업

기존 D1에는 이월 수기값이 0으로 저장되어 있으므로, v38 배포 후 월별 파일 보관함에서 전자랜드 2026년 5월 최종본의 `휴가 재반영`을 한 번 실행해야 합니다.

D1 화면에서 직접 수정할 필요는 없습니다. 새 컬럼은 API 최초 실행 시 자동 생성됩니다.
