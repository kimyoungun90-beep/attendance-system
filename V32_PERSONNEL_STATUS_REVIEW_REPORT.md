# v32 인력현황·연차대장 확인 요청 및 수기 변동 관리 보고서

## 1. 요청 반영 목적

매월 또는 인력 변동 발생 시 등록하는 인력현황을 연차 기준대장과 자동 대조하고, 누락·경로 불일치 인원을 별도 확인 대상으로 관리하도록 수정했습니다. 퇴사·경로이동·육아휴직 등의 결과를 관리자 화면 또는 최종본 엑셀에서 직접 입력할 수 있습니다.

## 2. 자동 확인 기준

- 인력현황에는 있으나 같은 경로의 연차대장에 없음: `연차대장 미등록`
- 연차대장에는 있으나 해당 월 인력현황에 없음: `인력현황 미등록`
- 같은 사번이 인력현황과 연차대장에 서로 다른 경로로 존재: `경로 불일치`
- 연차대장에 퇴사일이 있고 대상 월 시작 전에 퇴사한 인원: 자동 확인 대상에서 제외
- 경로 불일치는 이전 경로와 현재 경로를 각각 확인할 수 있도록 경로별 확인행으로 표시

## 3. 수기 처리 구분

- 확인 요청
- 재직·포함
- 퇴사
- 경로이동
- 육아휴직
- 기타휴직
- 제외

함께 저장되는 항목은 적용일, 종료일·복직일, 이동경로, 비고입니다. 적용기간이 입력된 퇴사·이동·휴직 상태는 이후 월에도 해당 기간 동안 자동으로 이어서 적용합니다.

`확인 요청`으로 되돌리면 기존 수기 확정값을 제거하고 다시 자동 확인 상태로 복귀합니다.

## 4. 인력현황 엑셀 자동 인식

기존 `랜드 인력DB`, `홈플 인력DB`, `매장매칭` 구조는 그대로 사용합니다. 다음 머리글이 존재하면 인력 변동값도 함께 읽습니다.

- 인력상태 / 재직상태 / 재직여부 / 상태 / 변동구분
- 퇴사일 / 퇴직일
- 적용일 / 변동일 / 이동일 / 휴직시작일
- 복직일 / 휴직종료일 / 종료일
- 이동경로 / 변경경로 / 전환경로 / 이동처
- 비고 / 휴퇴사일 / 휴·퇴사일

별도 시트명에 `인력 변동`, `인력 확인`, `퇴사`, `휴직`, `경로 이동`이 포함된 경우에도 사번과 처리구분을 찾아 읽습니다.

## 5. 최종본 엑셀 연동

자동 생성 최종본에 `인력 변동 확인` 시트를 추가했습니다.

- 상단 요약 카드
- 7행 고정
- 70% 확대 비율
- 눈금선 숨김
- 확인 유형 및 현재 인력현황·연차대장 경로 표시
- 처리구분, 적용일, 종료일·복직일, 이동경로, 비고를 직접 입력 가능

수정한 최종본을 `증빙 반영·비교 최종본`에 넣으면 다음을 동시에 읽습니다.

1. `출근 미등록` K열의 증빙 O
2. `인력 변동 확인` 시트의 퇴사·경로이동·육아휴직·휴직·제외 값

월 마감 저장 시 증빙 최종본에서 읽은 인력 변동값도 D1에 저장합니다.

## 6. 근태·연차 대조 반영

퇴사, 경로이동, 육아휴직, 기타휴직, 제외로 확정된 인원은 적용기간 동안 다음 자동 오류에서 제외됩니다.

- 인력현황에는 있으나 근무계획에 사번 없음
- 근무계획에는 있으나 인력현황에 사번 없음
- 월별 연차 신청서와 근무계획 대조

연차 잔여 자체를 임의로 다른 경로로 이전하지는 않습니다. 경로 이동자의 연차 적용기간과 기준입사일은 기존 연차 기준대장에서 수기로 확정해야 하며, 시스템은 경로 불일치와 이동일을 표시해 확인을 돕습니다.

## 7. D1 저장 정책

새 테이블:

`attendance_personnel_status_overrides`

기본키:

`대상 월 + 경로 + 사번`

저장 출처:

- `workforce`: 인력현황 엑셀에서 자동 인식
- `manual`: 관리자 화면에서 수기 저장
- `evidence`: 증빙 최종본의 인력 변동 확인 시트에서 반영

같은 월 인력현황을 교체 등록할 때 `workforce` 출처만 교체합니다. `manual`, `evidence` 출처는 임의 삭제하지 않습니다.
