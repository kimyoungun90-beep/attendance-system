# v31 월별 파일 보관함 통합 및 자동 분류 수정 보고서

## 1. 변경 이유

기존 v30에는 아래 기능이 서로 다른 화면에 나뉘어 있었습니다.

- 대체휴무·보상휴가 부여 화면 하단의 `이전 최종본 휴가 사용내역 저장`
- `월 마감 파일 보관함`의 파일 저장·다운로드 기능

최종본을 보관한 뒤 휴가 사용내역을 다시 별도 화면에서 등록해야 해 중복 작업처럼 보였고, 사용자가 `파일 구분`을 직접 선택해야 해서 저장 기준을 이해하기 어려웠습니다.

## 2. 통합 화면

상단 메뉴를 `월별 파일 통합 관리`로 변경하고 다음 기능을 한 화면에 모았습니다.

- 홈플러스 파일 선택 영역
- 전자랜드 파일 선택 영역
- 대상 월 공통 선택
- 파일 자동 분류 결과 미리보기
- 파일 저장·교체
- 최종본 대체휴무·보상휴가 자동 반영
- 저장 파일 다운로드·삭제
- 계획표/근태표 분석 화면 불러오기
- 최종본 증빙 반영·월 마감 비교
- 최종본 휴가내역 재반영

기존 `대체휴무·보상휴가 부여` 화면 아래에 있던 별도 최종본 가져오기 영역은 제거했습니다.

## 3. 홈플러스·전자랜드 분리

경로 드롭다운 하나를 바꿔가며 등록하는 방식 대신 아래 파일 입력 영역을 별도로 배치했습니다.

- `홈플러스` 파일
- `전자랜드` 파일

한쪽 경로 파일만 선택해 저장할 수도 있고, 같은 대상 월의 두 경로 파일을 동시에 선택해 순서대로 저장할 수도 있습니다. 각 경로에서 여러 파일을 선택할 수 있어 계획표·근태표·최종본을 한 번에 등록할 수 있습니다.

## 4. 파일 구분 자동화

화면의 수동 `파일 구분` 선택을 제거했습니다. 내부적으로는 기존 D1 교체 기준을 유지하기 위해 파일 종류 값을 계속 사용하지만, 사용자가 직접 고르지 않도록 변경했습니다.

자동 분류 우선순위는 다음과 같습니다.

1. 상담사근태 시트 또는 최종본·출퇴근현황·수정본 이름 → `최종본·수정본`
2. 근무계획·매장근무계획·스케줄 시트/파일명 → `계획표`
3. 근태관리·근태 RAW·출퇴근 RAW 시트/파일명 → `실제 근태표`
4. 위 조건에 해당하지 않음 → `기타`

같은 경로·같은 월·같은 자동 분류 파일을 다시 올리면 기존 파일을 교체하는 기존 정책은 유지됩니다.

## 5. 최종본 자동 연동

선택한 파일에 `상담사근태` 시트가 있으면 다음 순서로 처리합니다.

1. 파일 보관함에 최종본 저장·교체
2. 상담사근태 시트만 분석
3. 대체휴무·보상휴가 사용일과 인력 정보를 D1 누적 장부에 저장·교체
4. 저장 목록의 휴가내역 상태를 `반영 완료`로 표시

최종본 파일은 저장됐으나 상담사근태 구조 확인 또는 휴가내역 반영에 실패한 경우, 저장 목록에 `미반영`으로 표시됩니다. 이때 표의 `휴가내역 반영` 버튼으로 다시 처리할 수 있습니다. 이미 반영된 파일은 `휴가 재반영` 버튼으로 같은 경로·같은 월 자료를 교체 갱신할 수 있습니다.

## 6. 저장 목록 개선

저장 목록에 다음 항목을 표시합니다.

- 경로
- 대상 월
- 자동 분류
- 파일명
- 휴가내역 반영 상태
- 용량
- 저장 방식(D1/R2)
- 메모
- 저장일
- 관리 버튼

상단 요약에는 보관 파일 수, 홈플러스 파일 수, 전자랜드 파일 수, 최종본 휴가 반영 수, 전체 용량을 표시합니다.

## 7. 데이터 보호

- 신규 마이그레이션 없음
- 기존 D1 테이블 삭제 없음
- 기존 월 마감·연차·대체휴무·보상휴가 자료 삭제 없음
- 같은 경로·같은 월·같은 자동 분류에 해당하는 보관 파일만 기존 방식대로 교체
- 최종본 휴가 반영은 기존 `/api/final-leave-import` 교체 로직을 그대로 사용
