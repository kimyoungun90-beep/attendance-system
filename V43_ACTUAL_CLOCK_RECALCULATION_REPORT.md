- 배포 방식: Cloudflare Pages 자동 배포용 전체 덮어쓰기 ZIP

## 실제 업로드 자료 확인

전자랜드·홈플러스 근태 원본 모두 다음 헤더를 사용합니다.

- `이름`
- `근무일자`
- `(실제)출근시간`
- `사번`
- `(변경)출근시간`
- `출근지점`

근무계획 원본은 `매장명`, `사번`, `이름`, `재직상태`, `01일`~`31일` 헤더 구조입니다. v43은 A/B/C/D 같은 열 위치를 보조값으로 사용하지 않고, 실제 헤더명과 날짜 헤더로 열을 찾습니다.

## 수정 최종본 재등록

같은 회사·같은 월의 최종본을 비교 파일로 다시 등록하면 상담사근태 날짜 셀의 변경값을 자동 생성값과 비교합니다. 실제로 변경된 값만 확정값으로 반영합니다.

반영 대상:

- 출근시간 직접 수정
- `출근` 확정
- 연차·오전반차·오후반차
- 휴무·대체휴일·보상휴가
- 공가·경조·무급휴가 등 근태 코드
- 날짜 셀을 빈칸으로 수정한 경우

구버전 v42 최종본의 `출근` 표시는 원본 실제 출근시간과 동일한 정상 출근으로 간주하므로, 단순 재등록 시 실제 시간이 `출근` 문구로 되돌아가지 않습니다.

## 처리 완료 연동

일 단위 이상 건은 다음 키로 연결합니다.

- `직원번호 + 날짜 + 이상 유형`

월 단위 휴무 초과는 다음 키로 연결합니다.

- `직원번호 + 회사 + 기준월`

연동 대상:

- 출근 미등록 ↔ 계획&근태 상이
- 계획&근태 상이 ↔ 휴무 초과자

신규 v43 키가 존재하면 이상 유형이 다른 동일 날짜 건은 임의로 함께 삭제하지 않습니다. 과거 파일은 기존 직원번호+날짜 또는 직원번호 방식으로만 보조 호환합니다.

## 전체 재계산

확정값 반영 후 기존 분석 흐름을 처음부터 실행하여 다음 결과를 다시 생성합니다.

- 상담사근태
- 최종 요약본
- 매니저별 이상 근태
- 주 근태 확인자
- 계획&근태 상이 인원
- 출근 미등록
- 휴무 초과자
- 연차·대체휴무·보상휴가 사용량과 잔여

출근시간 확정은 `workedDates`와 일별 상태에 반영되므로 주 3회 이하·6회 이상도 다시 계산됩니다. 휴무·대체휴무·보상휴가 확정은 사용 이벤트와 잔여 계산에 다시 반영됩니다.

## 저장 구조

기존 `attendance_monthly_employee_facts.daily_statuses_json`, 근무일 배열, 휴가 이벤트 JSON을 그대로 사용합니다. 같은 회사·같은 월을 다시 저장하면 기존 월 마감 상세 자료를 삭제하고 새 확정 결과로 교체한 뒤 누적을 재계산합니다.

따라서 D1 컬럼이나 테이블 변경은 없습니다.
