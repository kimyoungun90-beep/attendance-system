# v44 처리 완료 양방향 연동·파일 제거 반영 보고서

## 기준

- 기준 코드: v43
- 결과 버전: v44
- 배포 방식: GitHub 전체 덮어쓰기용 ZIP / Cloudflare Pages 자동 배포
- D1 마이그레이션: 불필요

## 처리 완료 연동 변경

### 일 단위

`출근 미등록` K열 또는 `계획&근태 상이 인원` M열 중 어느 한쪽에 O를 입력하면 `사번 + 발생일`이 같은 모든 일 단위 이상 건을 함께 완료합니다. 이상 유형이 `missing_clock_in`, `mismatch` 등으로 달라도 동일 날짜면 연결됩니다.

출근 미입력 성격의 행은 어느 시트에서 O를 입력해도 출근 증빙으로 반영되어 주간 출근 횟수에도 포함됩니다.

재분석 후에는 완료 표시를 남겨 두지 않고 다음 시트의 대상 행에서 제거합니다.

- 출근 미등록
- 계획&근태 상이 인원
- 전체 요약본
- 매니저별 이상 근태

### 월 단위 휴무 초과

- 휴무·대체휴무·보상휴가 관련 계획&근태 상이 행에 O 입력 → 같은 `사번 + 회사 + 기준월` 휴무 초과 완료
- 휴무 초과자 R열에 O 입력 → 같은 사번·회사·월의 휴무 관련 계획&근태 상이 완료

따라서 두 방향 모두 연결됩니다.

## 파일 제거

메인 분석 화면의 계획표, 실제 근태표, 연차신청현황, 증빙 최종본 카드 우측 상단에 `파일 제거` 버튼을 추가했습니다. 제거 시 파일 input과 내부 상태를 함께 초기화하므로 잘못 선택한 파일이 분석에 남지 않습니다.

## 호환성

v43의 `planMismatchCompletedKeys`도 신규 `dailyCompletedKeys`와 합쳐 읽으므로 기존 최종본을 그대로 사용할 수 있습니다.

## 캐시

화면 버전 및 JavaScript 쿼리 버전을 v44로 변경해 Cloudflare 배포 후 기존 v43 브라우저 캐시와 분리합니다.
