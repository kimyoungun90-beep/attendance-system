# v34 연차 분리 입력·처리 워크플로·휴무초과 자동대체 수정 보고서

## 1. 작업 기준

v33을 기준으로 기존 근태 분석, 증빙 O, 대체휴무·보상휴가, 연차 누적, 인력 변동, 월 마감 및 D1 저장 기능을 유지하면서 아래 기능을 추가했습니다.

- 홈플러스·전자랜드 연차 파일의 물리적 업로드 위치 분리
- 최종본 엑셀의 처리·전달 O 입력 및 재반영
- 기본휴무 초과분의 대체휴무·보상휴가 자동대체
- 전월 휴무초과 대체 내역 표시
- 전체 요약본의 완료 건 제거
- 월별 연차 신청과 계획·실제근태 비교 시트 추가

## 2. 홈플러스·전자랜드 연차 입력 위치 분리

`연차 누적 관리` 화면에 다음 네 개의 독립된 입력 영역을 만들었습니다.

1. 홈플러스 기존 연차대장 등록·교체
2. 전자랜드 기존 연차대장 등록·교체
3. 홈플러스 월별 연차 신청현황
4. 전자랜드 월별 연차 신청현황

경로 선택 드롭다운 하나를 공용으로 사용하는 방식이 아니라, 파일 선택창과 저장 버튼 자체가 경로별로 분리되어 있습니다.

### 홈플러스 기준대장

- 현재 사용 중인 단일 상담사 연차관리대장을 인식합니다.
- 구분, 사번, 사원명, 입사일, 연차기준일, 발생, 사용 가능, 사용, 잔여, 적용기간, 퇴사·휴직·경로이동 내용을 읽습니다.
- 수식 문자열을 기준으로 판단하지 않고 파일에 저장된 셀 값을 읽습니다.
- 실제 제공된 홈플러스 대장 파싱 검증 결과:
  - 반영 92명
  - 제외 69명
  - 1년 미만 14명
  - 잔여 합계 1,162일

### 전자랜드 기준대장

- 기존 1년 이상자·1년 미만자 분리 시트 구조를 인식합니다.
- 시스템 표준 연차 누적 현황 구조도 함께 인식합니다.
- 발생, 누적 사용, 잔여, 사용기간, 촉진일 및 인력정보를 읽습니다.
- 동명이인 구분을 위해 이름 뒤 숫자가 붙은 표기도 유지합니다.
- 전자랜드 분리 시트는 구조 인식용 합성 데이터로 1년 이상자·미만자 분기 및 사번 인식을 검증했습니다.

각 기준대장을 다시 등록하면 해당 경로의 기준값만 마지막 파일 기준으로 교체하고, 반대 경로 자료와 월별 승인 누적자료는 삭제하지 않습니다.

## 3. 최종본 처리 확인 입력칸

| 시트 | 입력 열 | 입력값 | 결과 |
|---|---:|---|---|
| 출근 미등록 | K열 | O | 증빙 완료 |
| 계획&근태 상이 인원 | M열 | O | 처리 완료 |
| 휴무 초과자 | Q열 | O | 처리 완료 |
| 매니저별 이상 근태 | L열 | O | 전달 완료 |

처리상태 열은 O 입력 여부에 따라 자동으로 바뀌며, 완료 시 연녹색 계열로 표시합니다.

다음 분석 때 수정 최종본을 `증빙 반영·비교 최종본`으로 등록하면 다음 상태를 함께 읽습니다.

- 출근 미등록 증빙
- 계획&근태 상이 처리 완료
- 휴무 초과 처리 완료
- 매니저별 전달 완료

`전체 요약본`과 매니저 요청 목록에는 완료된 건을 제외하고 미처리 건만 다시 생성합니다.

## 4. 휴무초과 자동대체

기본휴무를 초과해 휴무로 입력된 일수는 아래 순서로 자동 충당합니다.

1. 해당 월 계획표에 직접 사용한 대체휴무·보상휴가를 먼저 차감
2. 남은 기본휴무 초과분에 대해 잔여 대체휴무를 우선 사용
3. 대체휴무가 부족하면 잔여 보상휴가 사용
4. 두 잔여량으로도 충당하지 못한 일수는 미대체로 계산

예시:

- 기본휴무 10일
- 실제 휴무 12일
- 사용 가능한 대체휴무+보상휴가 3.5일
- 초과 2일을 전부 자동대체
- 대체사용 2일, 미대체 0일

충당하지 못한 경우 판정은 `휴무초과 확인 요청 · 0.5일 미대체`처럼 표시하고 빨간색으로 강조합니다.

`휴무 초과자` 시트에는 아래 항목을 표시합니다.

- 당월 초과
- 당월 대체사용
- 당월 미대체
- 전월 초과
- 전월 대체사용
- 전월 미대체
- 판정
- 처리여부(O 입력)
- 처리상태

대체휴무를 우선 사용하고 보상휴가를 다음으로 사용하는 규칙은 서버 재계산과 엑셀 미리보기 계산에 동일하게 반영했습니다.

## 5. 월별 연차 등록 현황 및 일자

새 시트 `해당 월 연차 등록 현황 및 일자`를 생성하며, 서울·경인·충청·경북·경남·전라 순으로 구분합니다.

### 정상

- 계획: 연차 또는 동일 반차
- 신청서: 같은 날짜·같은 종류로 승인
- 실제 출근기록: 없음

위 세 조건이 모두 맞는 경우만 `동일`로 처리합니다.

### 확인 요청

- 계획 연차·반차, 신청서 없음
- 계획 연차·반차, 실제 출근 있음
- 계획 공백, 신청서 연차·반차 있음
- 계획 휴무·교육·근무 등, 신청서 연차·반차 있음

반차도 동일 기준으로 비교합니다. 정상 건을 제외한 모든 항목은 확인 요청으로 분류합니다.

## 6. 기존 기능 보존

다음 기능과 데이터는 삭제하거나 초기화하지 않았습니다.

- 기존 D1 월 마감 및 누적자료
- 출근 증빙 O 처리
- 대체휴무·보상휴가 부여 및 사용내역
- 연차 누적 원장
- 인력 변동 확인
- 같은 경로·같은 월 재등록 시 기존 월 자료 교체
- 전체 시트 70% 배율, 7행 고정, 눈금선 숨김

## 7. D1 변경

`attendance_monthly_summaries_v3`에 아래 네 컬럼을 추가합니다.

- `dayoff_replacement_used`
- `dayoff_replacement_shortage`
- `dayoff_replacement_substitute_used`
- `dayoff_replacement_compensation_used`

마이그레이션 파일: `migrations/0010_dayoff_replacement_workflow.sql`
