# 전자랜드 · 홈플러스 근태 관리 v12

기존 Cloudflare Pages·D1 구조를 유지하면서 **대체휴무 부여 인원 판정, 직원 검색·제외, 상담사근태 보고서 우측 집계 열**을 보완한 버전입니다.

- GitHub 저장소: `kimyoungun90-beep/attendance-system`
- 운영 주소: `https://attendance-system-wzz.pages.dev/`
- Cloudflare D1: `attendance-manager-db`

## v12 핵심 수정

### 1. 대체휴무 부여 인원이 0명으로 표시되던 문제

기존 버전은 부여 대상자를 해당 월의 **월 마감 사실자료**에서만 찾았습니다. 아직 월 마감이 저장되지 않은 6월·7월 부여 설정은 대상자가 0명으로 표시될 수 있었습니다.

v12는 등록된 `인력·매장매칭`의 직원과 입사일을 기준으로 대상자를 판정합니다.

- 발생 월과 정확히 일치하는 인력자료를 우선 사용
- 정확한 월이 없으면 가장 가까운 월의 인력자료 사용
- 발생 월 말일까지 입사한 직원만 해당 부여 건의 대상
- 직원의 입사일이 없으면 최초 인력자료 등장 월을 보조 기준으로 사용
- 월 마감 저장 전에도 적용 인원과 총 부여량 표시

예시 설정:

| 발생 월 | 1인당 부여 | 사용기간 |
|---|---:|---|
| 2026-05 | 1일 | 2026-05-01 ~ 2026-07-31 |
| 2026-06 | 2일 | 2026-05-01 ~ 2026-07-31 |
| 2026-07 | 1일 | 2026-05-01 ~ 2026-07-31 |

위 설정 결과:

- 5월 말 이전 입사자: 총 4일
- 6월 입사자: 총 3일
- 7월 입사자: 총 1일

사용 시작일이 5월이면 6월·7월 발생분도 해당 직원이 입사한 이후 월의 근태 분석에서 함께 사용할 수 있습니다.

### 2. 사번·이름으로 직원 선택

`대체휴무·보상휴가 부여` 화면에서 수기 사번 입력을 제거했습니다.

- 경로를 홈플러스·전자랜드로 먼저 구분
- 전체 등록된 인력자료에서 사번 또는 이름 검색
- 사번별 개별 부여 대상 선택
- 경로 일괄 부여 제외 대상 여러 명 선택
- 선택 결과에 이름·사번·점포 표시
- 수정 화면에서도 기존 제외자 유지

### 3. 상담사근태 우측 집계 열

월 날짜 영역 바로 뒤에 `총 등록 현황 수`가 추가됩니다. 30일인 달은 30일 열 다음, 31일인 달은 31일 열 다음에 표시됩니다.

`AU5`부터 다음 순서로 작성됩니다.

| 열 | 항목 | 내용 |
|---|---|---|
| AU | 총 사용 가능 휴무 | `기본휴무(추가휴무)` 형식. 예: `8(4.5)` |
| AV | 휴무 등록 개수 | 계획표의 `휴무` 수량 |
| AW | 대체 휴무 개수 | 기본 휴무 초과분과 표기 대체휴일을 합친 사용 수량 |
| AX | 보상 휴가 개수 | 보상휴가 사용 수량 |
| AY | 휴무 초과 | 기본 휴무 기준 초과 수량 |
| AZ | 대체 휴무 초과 | 부여 잔여보다 많이 사용한 수량 |
| BA | 보상 휴가 초과 | 부여 잔여보다 많이 사용한 수량 |

기존 출근 수정·교육·반차·연차·공가·무급휴무·경조사·총 일수·연차 신청·증빙·비고·정상 입력률은 `BB` 이후로 이동합니다.

## 기존 연차 관리 기능

- 전자랜드: 직원별 입사일 기준 연차 생성
- 홈플러스: 매년 1월 1일 기준 연차 생성
- 1년 미만자: 월 입사일 기준 월차 1개씩, 최대 11개
- 재직 1년 시점 연차 15개 생성
- 최초 연차대장 수기 수정본 재등록 시 마지막 파일로 완전 교체
- 월별 승인·반려 파일은 승인된 연차·반차만 차감
- 같은 경로·같은 월 재등록 시 마지막 파일 기준으로 교체
- 6개월·9개월 촉진 대상 별도 관리
- 이름 대신 연락처가 저장되지 않도록 사원명 열 검증
- 연차 누적 현황 보고용 엑셀 저장

## 최종 엑셀 시트 9개

1. `상담사근태`
2. `계획&근태 상이 인원`

4. `휴무 초과자`
5. `전체 요약본`
6. `해당 월 연차 등록 현황 및 일자`
7. `연차 누적 현황`
8. `근무 계획`
9. `근태 RAW`

## 배포 방법

1. ZIP 압축을 풉니다.
2. 폴더 안의 `public`, `functions`, `migrations`, `README.md`를 기존 저장소에 덮어씁니다.
3. GitHub Desktop Summary에 다음 문구를 입력합니다.

```text
v12 대체휴무 부여 인원 및 최종 엑셀 집계 개선
```

4. `Commit to main`
5. `Push origin`
6. Cloudflare Pages 배포가 `Success`인지 확인
7. 브라우저에서 `Ctrl + F5`로 새로고침

별도 D1 SQL 실행은 필요하지 않습니다.

## 배포 후 확인 순서

1. 상단 버전이 `v12`인지 확인
2. `시스템 수정`에 인력·매장매칭 파일이 등록돼 있는지 확인
3. `대체휴무·보상휴가 부여`에서 새로고침
4. 기존 5월·6월·7월 부여 설정의 `적용 인원`이 표시되는지 확인
5. 기존 부여 설정은 삭제하거나 다시 만들 필요 없이 그대로 사용
6. 6월 근태를 다시 분석하여 상담사근태 파일 저장
7. 날짜 마지막 열 뒤 `총 등록 현황 수` 확인
8. `AU5:BA5`가 요청한 순서로 생성되는지 확인

기존에 저장된 월 마감 기록의 계산값까지 즉시 다시 맞춰야 하는 경우 해당 월을 다시 분석한 뒤 `월 마감 교체 저장`하면 마지막 결과로 재계산됩니다.
