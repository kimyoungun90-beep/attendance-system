# 근태 관리 시스템 v34

## v34 핵심 변경

1. 홈플러스와 전자랜드의 연차 기준대장 등록 위치를 완전히 분리했습니다.
   - 홈플러스 기존 연차대장 등록·교체
   - 전자랜드 기존 연차대장 등록·교체
2. 월별 연차 신청현황도 경로별 전용 등록칸으로 분리했습니다.
   - 홈플러스 월별 연차 신청현황
   - 전자랜드 월별 연차 신청현황
3. 각 경로에서 현재 사용 중인 서로 다른 파일·시트 구조를 읽고, 나중에 같은 원본 양식을 수기로 수정해 다시 올리면 해당 경로 기준대장만 마지막 파일 기준으로 교체합니다.
4. 기준대장 입력 형식은 달라도 저장 이후 누적 연차 관리는 기존 시스템 표준 원장으로 계속 관리합니다.
5. 최종본 엑셀의 처리 확인 입력칸을 확대했습니다.
   - 출근 미등록: K열 증빙 O
   - 계획&근태 상이 인원: M열 처리 O
   - 휴무 초과자: Q열 처리 O
   - 매니저별 이상 근태: L열 전달 O
6. 처리한 최종본을 증빙 파일로 다시 등록하면 완료 상태를 읽어 `전체 요약본`과 매니저 전달 목록에는 미처리 건만 남깁니다.
7. 기본휴무 초과분은 남아 있는 대체휴무를 먼저 사용하고, 부족하면 보상휴가로 자동 대체합니다.
   - 당월 초과 / 당월 대체사용 / 당월 미대체
   - 전월 초과 / 전월 대체사용 / 전월 미대체
   - 대체·보상으로도 충당하지 못하면 `휴무초과 확인 요청`으로 빨간색 표시
8. `해당 월 연차 등록 현황 및 일자` 시트를 지역별로 생성합니다.
   - 계획 연차·반차 + 동일 신청 + 출근 없음만 `동일`
   - 계획 연차·반차인데 신청 없음
   - 계획 연차·반차인데 출근 기록 있음
   - 계획 공백인데 신청 있음
   - 계획 휴무·교육·근무 등인데 신청 있음
   위 항목은 모두 확인 요청으로 분류합니다.

## 운영 방법

### 연차 기준대장 최초 등록 또는 수기 수정 반영

- `연차 누적 관리` 화면에서 홈플러스와 전자랜드의 전용 업로드칸을 각각 사용합니다.
- 기존 파일의 발생·사용·잔여·적용기간·퇴사·휴직·경로이동 등을 수기로 수정한 경우 같은 전용 칸에 다시 등록합니다.
- 다시 등록하면 해당 경로 기준대장만 교체하며, 다른 경로와 월별 승인 누적자료는 유지합니다.

### 월별 연차 신청현황

- 홈플러스와 전자랜드 전용 월별 등록칸에 각각 파일을 올립니다.
- 같은 경로·같은 월을 다시 저장하면 해당 월 승인·반려 자료를 마지막 파일 기준으로 교체하여 중복 누적하지 않습니다.

### 처리 완료·전달 완료 반영

- 시스템이 만든 최종본에서 필요한 칸에 `O`를 입력해 저장합니다.
- 해당 파일을 다음 분석의 `증빙 반영·비교 최종본`으로 등록합니다.
- 완료된 계획 상이, 휴무 초과, 출근 증빙 및 매니저 전달 상태를 읽어 미처리 목록만 다시 표시합니다.
