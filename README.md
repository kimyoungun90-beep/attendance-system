# 근태 관리 시스템 v19


## v19 핵심 수정

- 월 마감 최종본 A/B 전용 비교 화면 추가: 사번·날짜별 차이만 표시하고 차이 엑셀 저장 가능
- 파일 보관함의 결과·수정본은 계획표 칸으로 보내지 않고 증빙 반영/기준본 A/비교본 B로 구분
- 최종본 파일명 자동 감지로 보관 구분을 결과·수정본으로 자동 선택
- 출근 미등록 시트의 `근태수정필요일` 머리글을 인식하여 K열 O 증빙을 정확히 반영
- 증빙 완료 건은 상담사근태에서 출근(#A9D08E)으로 표시되고 출근 미등록·계획&근태 상이·전체 요약본·매니저별 이상 근태에서 제거
- 대체휴무 발생일 자동 처리와 부여 대상을 분리: 지정일에 휴무+무출근은 기존 대체휴무 1일 사용, 실제 출근자는 해당 발생일 부여분 유지/획득
- 같은 월 수정본 재분석 시 실제 출근자 부여는 현재 업로드 파일 기준으로 다시 계산
- 기존 부여 기록의 발생일이 비어 있으면 안전하게 해당 월 1일로 보정되므로, 실제 발생일이 6월 3일이라면 배포 후 해당 부여 기록의 발생일을 `2026-06-03`으로 한 번 확인·수정해야 함


기존 Cloudflare Pages·D1 구조를 유지하면서, 근태 월 마감과 별도로 **연차 최초 기준대장 + 월별 승인·반려 파일**을 누적 관리하는 버전입니다.

- GitHub 저장소: `kimyoungun90-beep/attendance-system`
- 운영 주소: `https://attendance-system-wzz.pages.dev/`

## 핵심 구조

### 1. 연차 최초 기준대장

`연차 누적 관리` 메뉴에서 경로별 현재 연차대장을 처음 한 번 등록합니다.

- 전자랜드: 직원별 입사일 기준 연차 생성
- 홈플러스: 1월 1일 기준 연차 생성
- 1년 미만자: 월 입사일 기준 월차 1개씩, 최대 11개
- 재직 1년 시점: 연차 15개 생성 후 1년 이상자로 전환
- 발생·사용·잔여·사용기간·촉진일·퇴사일을 직원 사번 기준으로 저장
- 기준일이 포함된 `사용기간`을 찾아 직원별 현재 연차 묶음을 선택하므로, 아직 2026년 연차 발생일이 오지 않은 직원은 2025년 발생분을 현재 잔여로 가져옴
- 수기 수정한 초본을 다시 등록하면 해당 경로의 기존 기준자료를 완전히 교체

### 2. 월별 승인·반려 파일

매월 초 받은 `연차신청현황` 파일을 경로와 월별로 등록합니다.

- 승인된 연차·반차만 잔여 연차에서 차감
- 반려·취소·미사용·대기 건은 이력에는 남기되 잔여에서 차감하지 않음
- 같은 경로·같은 월 파일을 다시 등록하면 기존 월 자료를 완전히 교체
- 수정본을 재등록할 경우 마지막 파일이 최종 기준
- 근태 월 마감 시 선택한 연차신청현황도 해당 월 원장에 자동 저장
- 엑셀 내부 범위가 `A1`로 잘못 저장된 파일도 실제 셀 범위를 다시 찾아 전체 행을 읽음
- 선택한 월에 자료가 없으면 휴가일자가 가장 많은 실제 대상 월로 자동 보정

### 3. 촉진 대상 관리

- 연차 생성일 기준 6개월: 1차 촉진
- 연차 생성일 기준 9개월: 2차 촉진
- 촉진 예정일 31일 전부터 표시
- 촉진일로부터 1개월이 지나면 대상 목록에서 자동 제외
- 촉진 대상은 웹과 `연차 누적 현황` 시트에 별도 표시

### 4. 근태 판정

- 휴무·대체휴무·보상휴가·연차·공가 등인데 출근기록이 있는 사람
- 근무·근무A/B/C·교육·공백인데 출근기록이 없는 사람
- 계획과 실제 근태가 다른 사람
- 기본 휴무 초과자
- 대체휴무·보상휴가 사용량, 잔여, 초과 사용
- 계획에는 연차·반차가 있지만 신청내역이 없는 사람
- 신청은 있으나 계획과 일자가 다른 사람

## v11 수정 사항

- 사원명 열을 정확히 찾고 전화번호·이메일을 이름으로 저장하지 않도록 재검증
- 연차 누적 현황 화면에 `보고용 엑셀 저장` 버튼 추가
- 기존 오류 자료는 배포 후 최초 연차대장을 같은 파일로 다시 등록하면 148명 전체가 정상 이름으로 교체됩니다.


## v12 수정 사항

- `상담사근태` 시트의 `휴무` 합계를 근무계획 원본 개수가 아닌 **일별 셀에 최종 표시된 `휴무` 개수**로 계산합니다.
- 계획상 휴무였지만 실제 출근기록으로 일별 셀이 `출근`으로 표시되는 경우, 휴무 합계에서 제외됩니다.
- 열 이름을 `실제 표시 휴무`, `사용가능 휴무(기본+대체)`로 구분해 사용한 휴무와 사용 가능한 휴무가 혼동되지 않게 했습니다.
- 계획 휴무와 최종 표시 휴무가 다르면 비고에 두 수량을 함께 표시합니다.

## v13 수정 사항

- `상담사근태` 요약 열을 **해당 월의 마지막 날짜 바로 다음 열**부터 배치합니다.
  - 2026년 6월 기준: `AS5`부터 시작
  - 31일인 달은 일자 열이 하나 더 있으므로 `AT5`부터 시작
- 2026년 6월 기준 요약 열은 다음 순서입니다.
  - `AS`: 총 등록 현황
  - `AT`: 출근 등록 횟수
  - `AU`: 휴무 가능 개수 — `기본휴무(대체+보상 사용 가능 잔여)` 형식, 예: `8(4.5)`
  - `AV`: 휴무 사용 개수
  - `AW`: 휴무 초과 개수
  - `AX`: 대체+보상 휴무 개수
  - `AY`: 대체+보상 초과 개수
- 휴무 사용량은 근무계획 원본이 아니라 **최종 일별 표시값**으로 계산합니다.
- `AW 휴무 초과`가 있어도 대체휴무·보상휴가 사용 가능 잔여로 충당되면 빨간색으로 표시하지 않습니다.
- `AY 대체+보상 초과`가 실제로 0보다 큰 경우에만 빨간색으로 표시합니다.
- 6월에 대체휴무나 보상휴가를 사용하면 사용분이 누적 차감되어 7월의 `휴무 가능 개수` 괄호 안 숫자에 반영됩니다.



## v14 수정 사항

- 대체휴무·보상휴가 경로 일괄 부여 대상을 월 마감자료가 아니라 **인력·매장매칭의 입사일**로 판정합니다.
  - 5월 발생분: 5월 1일 이전 입사자
  - 6월 발생분: 6월 1일 이전 입사자
  - 7월 발생분: 7월 1일 이전 입사자
- 사용 시작일이 대상 월 이전이면, 발생 월이 미래여도 사용기간 안에서는 잔여에 포함합니다.
  - 예: 7월 발생분의 사용 시작일이 5월 1일이면 6월 분석에서도 사용 가능 잔여에 포함
- 기존 부여 현황의 `적용 인원 0명·총 부여 0일` 오류를 수정했습니다.
- 상담사근태 6월 기준 요약 열:
  - AS 총 등록 현황
  - AT 출근 등록 횟수
  - AU 휴무 가능 개수
  - AV 휴무 등록 개수
  - AW 휴무 초과 개수
  - AX 대체+보상 가능 개수
  - AY 대체+보상 등록 개수
  - AZ 대체+보상 초과 개수
- 휴무 집계 열은 파란색 계열, 대체·보상 집계 열은 보라색 계열로 구분합니다.
- 대체·보상 초과는 `휴무 초과 + 대체·보상 등록 - 대체·보상 가능`이 0보다 큰 경우에만 빨간색으로 표시합니다.


## v15 수정 사항

- 모든 엑셀 시트의 눈금선을 표시하지 않도록 저장합니다.
- `상담사근태` 제목·안내문·헤더·본문의 병합, 높이, 글꼴, 색상을 지정 양식으로 통일했습니다.
- B2:J2 제목, B3:J3·B4:J4 안내문 병합 및 서식을 적용했습니다.
- N5:N6을 병합하고 N열은 상담사 마지막 행까지 연한 노랑으로 표시합니다.
- O5부터 월 마지막 날짜까지 연한 파랑으로 표시하며 토·일요일과 2026년 공휴일은 빨간 글씨로 표시합니다.
- J열 성명은 상담사 마지막 행까지 노란색으로 표시합니다.
- 동일 사번이 월 중 점포 변경으로 두 번째 행부터 반복될 경우 I열 사번과 J열 성명을 분홍색 바탕·빨간 글씨로 표시합니다.
- 본문 행은 맑은 고딕 10pt, 높이 15.3pt로 통일했습니다.
- 휴무, 대체휴일, 보상휴가는 서로 다른 채우기 색으로 구분합니다.

## v16 긴급 수정 사항

- `XLSX.write` / `writeFile`에 지원되지 않는 `showGridLines` 쓰기 옵션을 제거했습니다.
- 모든 시트는 안전한 시트 보기 속성으로 눈금선 숨김을 적용합니다.
- 저장 직전 중복·겹침 병합 범위를 정리해 Excel, 한셀, 모바일 뷰어에서 열리지 않는 문제를 방지합니다.

배포 후에는 v15에서 내려받은 파일을 다시 사용하지 말고, 근태를 재분석한 뒤 최종본 엑셀을 새로 저장하십시오.


## v17 수정 사항

- 계획·근태 상이 및 미입력 일자 색상을 `#F33E0D` 배경과 흰색 글자로 통일했습니다.
- 보상휴가 정상 사용은 노란색(`#FFFF00`), 대체휴일 정상 사용은 회색(`#BFBFBF`)으로 표시합니다. 초과 사용 또는 오류 일자는 빨간색으로 표시합니다.
- 기본 휴무 초과와 대체·보상 초과를 분리했습니다. `AW`는 기본 휴무만, `AZ`는 실제 대체휴일·보상휴가 등록분만 계산합니다.
- 과거 월 마감에 저장된 `기본 휴무 초과` 가상 이벤트가 대체휴무 잔여를 차감하지 않도록 재계산에서 제외했습니다.
- 상담사근태 `B3:B4` 안내문은 15pt, 5행 높이는 33pt로 조정했습니다.
- 6월 기준 `AU:AW` 너비는 13.78, `AX:AZ` 너비는 10.89로 조정했습니다.


## v18 수정 사항

### 1. 발생일 기준 대체휴무 자동 부여·자동 사용

- `대체휴무·보상휴가 부여` 화면에서 발생일을 여러 개 입력할 수 있습니다.
  - 줄바꿈, 쉼표, 공백으로 구분
  - 입력한 날짜마다 별도 부여 기록 생성
- 대상자는 각 발생일 기준으로 판단합니다.
  - `입사일 ≤ 발생일`: 부여 대상
  - `입사일 > 발생일`: 제외
  - 예: 2026-06-03 발생분은 2026-06-05 입사자를 제외
- 대체휴무 발생일에 다음 조건을 모두 충족하면 자동 사용으로 처리합니다.
  - 근무계획이 `휴무`
  - 실제 출근시간과 변경 출근시간이 모두 없음
  - 해당 직원에게 그 날짜의 대체휴무 부여분이 있음
- 자동 사용 시:
  - 상담사근태 일별 값은 `휴무` 유지
  - 기본 휴무 등록 개수에 1일 포함
  - 대체휴무 잔여에서 1일 차감
  - 실제 근태 열에 다른 문구가 있어도 출근시간이 없으면 지정일 휴무를 우선
- 근무계획은 `휴무`지만 출근시간이 있으면:
  - 대체휴무를 사용하지 않음
  - 잔여 1일 유지 또는 기존 차감 복원
  - `계획·실제 불일치`에는 계속 표시
- 계획에 이미 `대체휴일(1일/0.5일)`이 명시된 경우 자동 사용을 중복 추가하지 않습니다.
- 부여 기록을 삭제하거나 발생일을 변경하면 기존 월 마감 재계산 시 더 이상 유효하지 않은 자동 차감은 제거됩니다. 새로 추가한 발생일의 자동 사용은 해당 월을 다시 분석하고 월 마감 교체 저장해야 반영됩니다.

### 2. 출근 증빙 O 자동 반영

1. 기존 최종본의 `출근 미등록` 시트에서 `증빙여부(O 입력)` 열에 `O`를 입력합니다.
2. 근태 종합 점검의 `증빙 반영·비교 최종본`에 그 파일을 선택합니다.
3. 계획표·실제 근태표와 함께 다시 분석합니다.

반영 결과:

- 상담사근태 해당 일자를 `출근`으로 변경
- 셀 채우기 `#A9D08E`, 검은색 굵은 글씨
- `출근 미등록`에서 제외
- `계획&근태 상이 인원`의 `근무인데 출근기록 없음`에서 제외
- `전체 요약본` 미해결 건수에서 제외
- 웹의 매니저별 이상 근태와 엑셀 `매니저별 이상 근태`에서 제외
- 월 마감의 실제 출근일에도 저장되어, 이후 `특정일 실제 출근자만 부여` 판정에도 출근으로 인정

단, 계획이 휴무인데 실제 출근한 것처럼 **출근 증빙 후에도 실제 계획 불일치가 남는 경우**는 계획 상이에서 유지합니다.

### 3. 매니저별 상담사 이상 근태

- 웹 결과에 `매니저 수정요청` 탭과 매니저 선택 필터를 제공합니다.
- 근태 미입력, 휴무·휴가 출근, 계획 불일치, 휴무 초과, 대체·보상 초과, 연차 신청 불일치를 직원별로 묶습니다.
- 최종 엑셀에 `매니저별 이상 근태` 시트를 생성합니다.
- 증빙 완료된 일반 출근 미등록 건은 재분석 시 자동 제외됩니다.

### 4. 인력 매칭 메뉴명 변경

- 상단 `시스템 수정` 메뉴를 `인력 매칭`으로 변경했습니다.
- 기능과 월별 전체교체 정책은 그대로 유지합니다.

### 5. 월 마감 파일 보관함 교체 정책

- 수동 업로드도 같은 `경로 + 대상 월 + 파일 구분`이면 기존 파일을 삭제하고 마지막 업로드 파일로 교체합니다.
- 예: 전자랜드 2026-06 `결과·수정본`을 다시 올리면 이전 결과·수정본은 삭제되고 새 파일 1개만 남습니다.
- 다른 월, 다른 경로, 다른 파일 구분의 자료는 유지됩니다.
