# 전자랜드 · 홈플러스 근태 관리 v11

기존 Cloudflare Pages·D1 구조를 유지하면서, 근태 월 마감과 별도로 **연차 최초 기준대장 + 월별 승인·반려 파일**을 누적 관리하는 버전입니다.

- GitHub 저장소: `kimyoungun90-beep/attendance-system`
- 운영 주소: `https://attendance-system-wzz.pages.dev/`
- Cloudflare D1: `attendance-manager-db`

## v11 핵심 구조

### 1. 연차 최초 기준대장

`연차 누적 관리` 메뉴에서 경로별 현재 연차대장을 처음 한 번 등록합니다.

- 전자랜드: 직원별 입사일 기준 연차 생성
- 홈플러스: 1월 1일 기준 연차 생성
- 1년 미만자: 월 입사일 기준 월차 1개씩, 최대 11개
- 재직 1년 시점: 연차 15개 생성 후 1년 이상자로 전환
- 발생·사용·잔여·사용기간·촉진일·퇴사일을 직원 사번 기준으로 저장
- 기준일이 포함된 `사용기간`을 찾아 직원별 현재 연차 묶음을 선택하므로, 아직 2026년 연차 발생일이 오지 않은 직원은 2025년 발생분을 현재 잔여로 가져옴
- 수기 수정한 초본을 다시 등록하면 해당 경로의 기존 기준자료를 완전히 교체

### 2. 월별 승인·반려 파일

매월 초 받은 `연차신청현황` 파일을 경로와 월별로 등록합니다.

- 승인된 연차·반차만 잔여 연차에서 차감
- 반려·취소·미사용·대기 건은 이력에는 남기되 잔여에서 차감하지 않음
- 같은 경로·같은 월 파일을 다시 등록하면 기존 월 자료를 완전히 교체
- 수정본을 재등록할 경우 마지막 파일이 최종 기준
- 근태 월 마감 시 선택한 연차신청현황도 해당 월 원장에 자동 저장
- 엑셀 내부 범위가 `A1`로 잘못 저장된 파일도 실제 셀 범위를 다시 찾아 전체 행을 읽음
- 선택한 월에 자료가 없으면 휴가일자가 가장 많은 실제 대상 월로 자동 보정

### 3. 촉진 대상 관리

- 연차 생성일 기준 6개월: 1차 촉진
- 연차 생성일 기준 9개월: 2차 촉진
- 촉진 예정일 31일 전부터 표시
- 촉진일로부터 1개월이 지나면 대상 목록에서 자동 제외
- 촉진 대상은 웹과 `연차 누적 현황` 시트에 별도 표시

### 4. 근태 판정

- 휴무·대체휴무·보상휴가·연차·공가 등인데 출근기록이 있는 사람
- 근무·근무A/B/C·교육·공백인데 출근기록이 없는 사람
- 계획과 실제 근태가 다른 사람
- 기본 휴무 초과자
- 대체휴무·보상휴가 사용량, 잔여, 초과 사용
- 계획에는 연차·반차가 있지만 신청내역이 없는 사람
- 신청은 있으나 계획과 일자가 다른 사람

### 5. 최종 엑셀 시트 9개

1. `상담사근태`
2. `계획&근태 상이 인원`
3. `출근 미등록`
4. `휴무 초과자`
5. `전체 요약본`
6. `해당 월 연차 등록 현황 및 일자`
7. `연차 누적 현황`
8. `근무 계획`
9. `근태 RAW`

`상담사근태` 우측에는 다음 항목이 자동 작성됩니다.

- 근무자입력(출근 횟수)
- 휴무
- 사용가능휴무
- 휴무 초과 개수
- 대체 휴무 이월 개수
- 출근 수정
- 교육
- 반차
- 연차
- 공가
- 무급휴무
- 경조사
- 총 일수
- 연차 미신청
- 연차 신청(승인)
- 출근 증빙
- 수정 완료
- 비고
- 정상 입력률

## 사용 순서

1. 관리자 로그인
2. `시스템 수정`에서 대상 월 인력·매장매칭 등록
3. `연차 누적 관리`에서 경로별 최초 연차대장 등록
4. 매월 초 `연차 누적 관리`에서 전월 승인·반려 파일 등록
5. `근태 종합 점검`에서 계획표와 근태 RAW 선택
6. 아직 월별 원장에 넣지 않은 연차신청현황이 있으면 함께 선택
7. 근태 종합 분석
8. 9개 시트 최종본 저장
9. 수기 수정이 필요하면 수정본을 파일 보관함에 등록
10. 확정 후 월 마감 교체 저장

## 배포 방법

1. ZIP 압축을 풉니다.
2. 폴더 안의 `public`, `functions`, `migrations`, `README.md`를 기존 저장소에 덮어씁니다.
3. GitHub Desktop Summary에 아래 문구를 입력합니다.

```text
v10 연차 최초대장 및 월별 승인반려 누적관리 추가
```

4. `Commit to main`
5. `Push origin`
6. Cloudflare Pages 배포가 `Success`인지 확인합니다.

## D1

`functions/_lib/schema.js`가 연차 원장 테이블을 자동 생성합니다. 신규 설치용 SQL은 `migrations/0006_annual_leave_ledger.sql`에도 포함되어 있습니다. 일반적인 기존 운영 배포에서는 별도 SQL 실행 없이 첫 API 호출 시 생성됩니다.

## 배포 후 확인

1. 상단 메뉴에 `연차 누적 관리`가 보이는지 확인
2. 최초 전자랜드 연차대장을 등록했을 때 1년 이상자와 1년 미만자가 분리되는지 확인
3. 같은 초본을 수정 후 재등록했을 때 기존 기준값이 교체되는지 확인
4. 월별 파일에서 승인 건만 잔여 연차에서 차감되는지 확인
5. 반려 건이 `해당 월 연차 등록 현황 및 일자`에 남고 잔여에서는 차감되지 않는지 확인
6. 동일 월 수정본 재등록 시 이전 월 자료가 중복 누적되지 않는지 확인
7. 6개월·9개월 촉진 대상이 웹과 엑셀에 표시되는지 확인
8. 최종 엑셀 시트가 지정된 9개만 생성되는지 확인
9. `출근 미등록` K열에 O 입력 시 `상담사근태`의 해당 날짜가 출근으로 바뀌는지 확인


## v11 수정 사항

- 최초 연차대장 등록 시 이름 대신 연락처가 저장되는 문제 수정
- 시트 시작 열을 A1 기준으로 보정해 실제 엑셀 열 위치 유지
- 사원명 열을 정확히 찾고 전화번호·이메일을 이름으로 저장하지 않도록 재검증
- 연차 누적 현황 화면에 `보고용 엑셀 저장` 버튼 추가
- 기존 오류 자료는 배포 후 최초 연차대장을 같은 파일로 다시 등록하면 148명 전체가 정상 이름으로 교체됩니다.
