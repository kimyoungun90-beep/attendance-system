# 근태 관리 시스템 v32

## v32 핵심

1. 월별 인력현황과 홈플러스·전자랜드 연차 기준대장을 사번 기준으로 자동 대조합니다.
2. 인력현황에만 있는 사람, 연차대장에만 있는 사람, 두 자료의 경로가 다른 사람을 `인력 확인 요청`으로 표시합니다.
3. 관리자 화면에서 `재직·포함 / 퇴사 / 경로이동 / 육아휴직 / 기타휴직 / 제외`를 수기로 지정하고 적용일·종료일·이동경로·비고를 저장할 수 있습니다. 적용기간이 있으면 이후 월에도 자동 승계합니다.
4. 인력현황 파일의 상태·퇴사일·이동일·휴직기간 열과 별도 `인력 변동/퇴사/휴직/경로이동` 시트를 자동 인식합니다.
5. 자동 생성 최종본에 `인력 변동 확인` 시트를 추가했습니다. 이 시트를 수정한 뒤 기존 `증빙 반영·비교 최종본`에 다시 넣으면 출근 증빙 O와 인력 변동을 동시에 인식합니다.
6. 퇴사·경로이동·육아휴직·기타휴직·제외로 확정된 인원은 적용기간 동안 월별 인력 누락 및 연차 신청 대조 오류에서 제외합니다.
7. 같은 월 인력현황을 다시 등록하면 인력현황에서 읽은 변동정보만 교체하며, 관리자가 수기로 저장한 상태와 증빙 최종본에서 반영한 상태는 유지합니다.
8. 기존 월 마감, 연차 누적, 대체휴무·보상휴가, 증빙 O, D1 자료 및 같은 경로·같은 월 교체 저장 방식은 유지합니다.

배포 시 `migrations/0009_personnel_status_review.sql`을 D1에 적용하거나, 앱 최초 API 호출 시 자동 스키마 생성을 사용하면 됩니다.

상세 내용은 `V32_PERSONNEL_STATUS_REVIEW_REPORT.md`를 확인하세요.
