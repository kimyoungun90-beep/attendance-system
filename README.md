# 전자랜드 · 홈플러스 근태 관리 v6

기존 운영 주소와 Cloudflare D1을 그대로 사용하면서, 매월 인력·매장매칭 파일을 기준으로 상담사근태 최종본을 자동 생성하는 버전입니다.

- GitHub 저장소: `kimyoungun90-beep/attendance-system`
- 운영 주소: `https://attendance-system-wzz.pages.dev/`
- Cloudflare D1: `attendance-manager-db`

## v6 핵심 변경

### 1. 시스템 수정 → 월별 인력·매장매칭 등록

매월 받은 `인력 및 매장매칭(랜드,홈플).xlsx`를 등록합니다.

자동 인식 시트:

- `랜드 인력DB`
- `홈플 인력DB`
- `매장매칭`

같은 월 파일을 다시 저장하면 해당 월 자료를 완전히 교체합니다. 원본 파일도 D1 또는 연결된 R2에 함께 보관되어 다시 내려받을 수 있습니다.

### 2. 상담사근태 B~J 자동 작성

등록한 월별 인력·매장매칭 자료를 기준으로 `O월 상담사근태` 6행 머리글 아래의 B~J를 자동 작성합니다.

- B: 지역장
- C: 매니저
- D: 지역1
- E: 지역2
- F: 매장코드
- G: 매장명
- H: 포탈사번
- I: 사번
- J: 성명

포탈사번은 최초 등록 시 이전 월 최종본을 참고 파일로 함께 넣으면 `사번 ↔ 포탈사번`이 저장됩니다. 이후 월에는 저장된 포탈사번을 자동으로 사용합니다.

### 3. 이전 최종본 양식 그대로 다운로드

분석 결과에서 `최종본 양식 엑셀 저장`을 누르면 기존 공유용 최종본 모양을 기준으로 다음 시트를 생성합니다.

- `O월 상담사근태`
- `보상휴가(0.5일) 반영 인원`
- `증빙(필수기입)`
- `O월 근무계획`
- `O월 연차사용`
- `근태RAW`
- `연차RAW`
- `교육RAW`

월별 일수와 요일은 자동 변경됩니다.

### 4. 이상 날짜 빨간색 표시

근무계획에서 조금이라도 확인이 필요한 날짜는 빨간색으로 표시하고, 해당 직원 바로 아래 줄에 `빨간색 체크 사유(확인 필요)`를 날짜별로 작성합니다.

- 근무·교육·반차·공백인데 출근기록 없음
- 휴무·휴가·연차·대체휴일·보상휴가인데 출근기록 있음
- 계획과 실제 근태 불일치
- 기타 또는 미등록 코드
- 기본 휴무 초과
- 대체휴무·보상휴가 잔여 부족
- 인력·매장매칭과 근무계획의 사번 누락

### 5. `546` 공백 처리

계획표 또는 근태표의 빈 칸이 `546`으로 읽히는 경우 실제 코드나 출근시간으로 보지 않습니다.

- 계획표 `546` → 공백 계획 → 출근 필요
- 출근시간 `546` → 출근기록 없음
- 기타 정보 `546` → 빈 값

### 6. 기존 누적 기능 유지

- 경로별 월 마감 1건 저장 및 완전 교체
- 과거 월 교체 시 이후 월 전체 재계산
- 연차·반차 누적
- 대체휴무·보상휴가 0.5일 단위 부여
- 경로 일괄 또는 사번별 부여
- 특정 날짜 실제 출근자만 부여
- 사번별 제외
- 사용기한 내 이월, 종료 후 만료, 초과 사용 판정
- 월 마감 계획표·근태표·최종본 자동 보관

## 사용 순서

1. 관리자 로그인
2. `시스템 수정`에서 적용 월과 인력·매장매칭 파일 저장
3. 최초 1회만 이전 최종본을 포탈사번 참고 파일로 선택
4. `근태 종합 점검`에서 경로와 대상 월 선택
5. 근무계획 파일과 근태관리 파일 업로드
6. `근태 종합 분석`
7. 결과 확인 후 `최종본 양식 엑셀 저장`
8. 필요하면 엑셀을 수정한 뒤 `월 마감 파일 보관함`에 결과·수정본으로 저장
9. 확정 후 `월 마감 교체 저장`

## 배포 방법

1. ZIP 압축을 풉니다.
2. 압축을 푼 폴더 안의 `public`, `functions`, `migrations`, `README.md`를 복사합니다.
3. GitHub Desktop에서 `Repository → Show in Explorer`를 누릅니다.
4. 기존 `attendance-system` 폴더 안에 붙여넣고 `대상 폴더의 파일 바꾸기`를 누릅니다.
5. GitHub Desktop Summary에 아래 문구를 입력합니다.

```text
v6 월별 인력매칭 및 최종본 양식 자동 생성
```

6. `Commit to main`
7. 상단 `Push origin`
8. Cloudflare Pages 배포가 `Success`인지 확인합니다.

## D1 SQL 실행 여부

API가 필요한 테이블을 자동 생성하므로 일반적으로 SQL을 따로 실행하지 않아도 됩니다.

`migrations/0005_workforce_matching_and_final_template.sql`은 신규 DB 또는 수동 점검용입니다.

## 배포 후 확인

1. `시스템 수정` 메뉴가 보이는지 확인
2. 인력·매장매칭 파일 등록 후 전자랜드/홈플러스 인원이 표시되는지 확인
3. 이전 최종본 참고 파일 등록 후 포탈사번 인원이 표시되는지 확인
4. 해당 월·경로 분석 시 인력 매칭 등록 상태가 초록색으로 표시되는지 확인
5. 결과 엑셀의 `O월 상담사근태` B~J가 매칭 파일 기준으로 채워지는지 확인
6. 이상 날짜가 빨간색이며 `O월 근무계획` 바로 아래 줄에 사유가 적히는지 확인
7. `546`이 출근시간으로 오인되지 않는지 확인
8. 같은 월 인력 파일과 월 마감을 다시 저장했을 때 기존 자료가 교체되는지 확인
