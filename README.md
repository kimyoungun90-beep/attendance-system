# 전자랜드 · 홈플러스 근태 관리 시스템 v1

## 현재 구현 범위
- 회사 선택: 홈플러스 / 전자랜드
- 계획표와 실제 근태표 엑셀 업로드
- 파일명 대신 엑셀 머리글 자동 인식
- 사번 + 날짜 기준 출근 기록 비교
- 기본 판정: 계획표가 `근무`, `출근`, `공백`인데 실제/변경 출근시간이 없는 경우
- 넓은 판정: 휴무·휴가 키워드가 아닌 모든 계획 검사
- 파일 회사 감지 및 사번 매칭률 경고
- 누락자 검색, 매장 필터, 엑셀 저장
- 월 마감 결과 D1 영구 저장 또는 브라우저 로컬 저장
- 추후 휴무 기준·대체휴무 기능을 위한 D1 테이블 포함

## 폴더 구조
- `index.html`: 메인 화면
- `assets/app.js`: 엑셀 판독 및 비교 로직
- `assets/styles.css`: 화면 디자인
- `functions/api`: 로그인 및 월 마감 저장 API
- `migrations/0001_init.sql`: D1 테이블 생성 SQL
- `wrangler.toml`: Cloudflare Pages 설정 예시

## Cloudflare 배포 순서
1. GitHub 새 저장소에 이 폴더의 파일을 그대로 업로드합니다.
2. Cloudflare Pages에서 해당 저장소를 연결합니다.
3. 빌드 명령은 비워두고, 출력 디렉터리는 `.` 로 설정합니다.
4. Cloudflare D1에서 `attendance-manager-db` 데이터베이스를 생성합니다.
5. D1 콘솔에서 `migrations/0001_init.sql` 전체를 실행합니다.
6. Pages 프로젝트 > Settings > Bindings에서 D1 바인딩을 추가합니다.
   - Variable name: `DB`
   - D1 database: 방금 만든 데이터베이스
7. Pages 프로젝트 환경 변수/Secret에 아래 값을 추가합니다.
   - `ADMIN_PASSWORD`: 로그인에 사용할 비밀번호
   - `SESSION_SECRET`: 영문/숫자/특수문자를 섞은 긴 임의 문자열
8. 재배포합니다.

## 파일 형식 인식 기준
### 계획표
- 필수: `매장명`, `사번`, `이름`
- 일자: `01일`부터 `31일`
- 선택: `재직상태`

### 실제 근태표
- 필수: `사번`, `근무일자`
- 출근시간: `(실제)출근시간` 또는 `(변경)출근시간`
- 선택: `이름`, `출근지점`
Cloudflare D1 설정 완료

## 데이터 보안
- 엑셀 파일 자체는 브라우저에서 읽고 서버로 업로드하지 않습니다.
- 월 마감 저장 시에는 비교 결과와 파일명만 D1에 저장합니다.
- D1 API는 관리자 비밀번호 로그인 후에만 사용할 수 있습니다.
