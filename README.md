# 전자랜드 · 홈플러스 근태 관리 v8

기존 운영 주소와 Cloudflare D1을 그대로 사용하면서, 월별 인력·매장매칭·근무계획·근태관리·연차신청현황을 한 번에 대조하고 보고용 최종본을 생성하는 버전입니다.

- GitHub 저장소: `kimyoungun90-beep/attendance-system`
- 운영 주소: `https://attendance-system-wzz.pages.dev/`
- Cloudflare D1: `attendance-manager-db`

## v8 핵심 변경

### 1. 증빙 O 입력 시 상담사근태 자동 반영

보고용 최종본의 `증빙(필수기입)` 시트 K열 `증빙여부(O 입력)`에 `O`를 입력하면, 같은 사번·같은 날짜의 `O월 상담사근태` 값이 자동으로 바뀝니다.

- 기존 값이 `미입력`인 경우 → `출근`
- 다른 정상 근태 값은 임의로 변경하지 않음
- 사번과 발생일을 함께 비교하므로 동명이인 오반영 방지

### 2. 상담사근태 상단 보고용 재정리

- B2:BL2: 제목과 대상 인원 통합
- B3:BL3: 증빙 O 자동반영 안내 문구 병합
- B4:BL4: 계획·근태·연차 불일치 안내 문구 병합
- 상단 색상을 네이비·라이트블루 계열로 통일
- 날짜를 실제 날짜 형식 `yyyymmdd`로 저장
- 날짜 열 너비를 확대하여 `#######` 표시 방지
- 상단 6행과 B~J 직원정보 열 고정

### 3. 매니저 수정요청 멘트 자동 생성

분석 결과와 연차 신청 대조 결과를 직원별로 모아서 `매니저 수정요청` 시트를 생성합니다.

예시:

```text
김영언 매니저님, 대구점 홍길동 상담사(E123456)의 6월 2일·7일 휴무이나 출근 기록이 있고, 6월 21일 기본 휴무 1일 초과 항목이 확인됩니다. 근무계획·근태·연차 신청내역 확인 후 수정 바랍니다.
```

포함 항목:

- 근무인데 근태 미입력
- 휴무·휴가인데 출근 기록 있음
- 계획과 실제 근태 불일치
- 기본 휴무 초과
- 대체휴무·보상휴가 초과 사용
- 인력·매장매칭 사번 누락
- 연차 신청과 근무계획 불일치
- 계획에는 연차·반차가 있으나 신청내역 없음

### 4. 연차신청현황 자동 대조

근태 종합 점검 화면에서 `연차신청현황` 파일을 추가로 선택할 수 있습니다.

대조 기준:

- 사번 + 휴가일자
- 신청 1일 ↔ 근무계획 `연차`
- 신청 0.5일 ↔ 근무계획 `오전반차` 또는 `오후반차`

결과 구분:

- 일치
- 신청 있음 · 계획 불일치
- 계획 있음 · 신청내역 없음
- 계획표에 사번 없음

결과 엑셀에는 `연차 신청 대조` 시트가 추가됩니다.

### 5. 다른 작성자의 최종본 비교

근태 종합 점검 화면에서 `다른 작성자의 최종본`을 추가로 선택할 수 있습니다.

동일 월 파일인 경우:

- 사번 + 날짜 기준으로 자동 생성값과 비교 파일 값을 대조
- 직원 또는 날짜 누락 표시
- 근태 값이 다른 직원·날짜·두 파일의 값을 한 행에 표시

월이 다른 경우:

- 직원별 날짜 값은 직접 비교하지 않음
- `월 불일치`로 표시
- 시트 구성과 역할을 기준으로 구조 비교만 수행

결과 엑셀에는 `최종본 비교` 시트가 추가됩니다.

### 6. 보고용 최종본 시트 구성

- `오류 요약`
- `매니저 수정요청`
- `연차 신청 대조`
- `최종본 비교`
- `O월 상담사근태`
- `보상휴가(0.5일) 반영 인원`
- `증빙(필수기입)`
- `O월 근무계획`
- `O월 연차사용`
- `근태RAW`
- `연차RAW`
- `교육RAW`

### 7. 기존 누적 기능 유지

- 경로별 월 마감 1건 저장 및 완전 교체
- 과거 월 교체 시 이후 월 전체 재계산
- 연차·반차 누적
- 대체휴무·보상휴가 0.5일 단위 부여
- 경로 일괄 또는 사번별 부여
- 특정 날짜 실제 출근자만 부여
- 사번별 제외
- 사용기한 내 이월, 종료 후 만료, 초과 사용 판정
- 월 마감 계획표·근태표·연차신청현황·비교 최종본·최종 결과 자동 보관

## 사용 순서

1. 관리자 로그인
2. `시스템 수정`에서 적용 월의 인력·매장매칭 파일 등록
3. `근태 종합 점검`에서 경로와 대상 월 선택
4. 근무계획 파일과 근태관리 파일 선택
5. 필요하면 연차신청현황 파일 선택
6. 필요하면 다른 작성자의 최종본 파일 선택
7. `근태 종합 분석`
8. 오류 요약·연차 신청 대조·매니저 수정요청·최종본 비교 확인
9. `최종본 양식 엑셀 저장`
10. 증빙 확인 후 `증빙(필수기입)` K열에 O 입력
11. 확정 후 월 마감 교체 저장

## 배포 방법

1. ZIP 압축을 풉니다.
2. 압축을 푼 폴더 안의 `public`, `functions`, `migrations`, `README.md`를 복사합니다.
3. GitHub Desktop에서 `Repository → Show in Explorer`를 누릅니다.
4. 기존 `attendance-system` 폴더 안에 붙여넣고 `대상 폴더의 파일 바꾸기`를 누릅니다.
5. GitHub Desktop Summary에 아래 문구를 입력합니다.

```text
v8 증빙 자동반영 및 연차·최종본 비교 추가
```

6. `Commit to main`
7. 상단 `Push origin`
8. Cloudflare Pages 배포가 `Success`인지 확인합니다.

## D1 SQL 실행 여부

이번 버전은 기존 DB 구조를 그대로 사용하므로 별도의 SQL 실행이 필요하지 않습니다.

## 배포 후 확인

1. 근태 종합 점검에 `연차신청현황`과 `다른 작성자의 최종본` 선택 영역이 보이는지 확인
2. 분석 결과에 `연차 신청 대조`, `매니저 수정요청`, `최종본 비교` 탭이 보이는지 확인
3. 최종 엑셀의 상담사근태 상단 B2~B4가 병합되고 색상이 통일됐는지 확인
4. 날짜가 `#######`이 아니라 `20260601` 형식으로 보이는지 확인
5. 증빙 시트 K열에 O를 입력했을 때 상담사근태의 해당 `미입력`이 `출근`으로 바뀌는지 확인
6. 동일 월 비교 파일에서는 사번+날짜별 불일치가 나타나는지 확인
7. 다른 월 비교 파일에서는 월 불일치와 구조 비교만 나타나는지 확인
8. 연차 신청 1일·0.5일이 각각 연차·반차 계획과 올바르게 대조되는지 확인
