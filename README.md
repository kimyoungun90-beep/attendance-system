# 전자랜드 · 홈플러스 근태 관리 v4

기존 운영 주소와 Cloudflare D1을 그대로 사용하면서 오류 수정과 월 마감 파일 보관함을 추가한 버전입니다.

- GitHub 저장소: `kimyoungun90-beep/attendance-system`
- 운영 주소: `https://attendance-system-wzz.pages.dev/`
- Cloudflare D1: `attendance-manager-db`

## 이번 수정 사항

### 1. 사번 인식 복구·강화

- 엑셀 첫 번째 시트만 보지 않고 통합문서의 모든 시트를 확인합니다.
- 계획표의 기존 고정 구조도 인식합니다.
  - A열 매장명
  - B열 사번
  - D열 이름
  - I열 이후 일자별 계획
- 실제 근태표의 기존 고정 구조도 인식합니다.
  - A열 이름
  - B열 근무일자
  - C열 출근시간
  - D열 사번
- `E248024`, 숫자형 사번, 공백·하이픈·엑셀의 `.0`이 붙은 사번을 정리해서 비교합니다.
- 머리글이 위쪽 여러 줄에 있거나 날짜 머리글이 두 줄인 파일도 검색합니다.

### 2. 대체휴무 설정 오류 수정

- 경로별 공통 대체휴무 API를 호출할 때 필요한 D1 테이블과 컬럼을 자동 점검하고 생성합니다.
- 이전 마이그레이션이 일부만 적용된 상태여도 다시 사용할 수 있도록 보완했습니다.
- 월 마감이 아직 없어도 공통 대체휴무 설정 자체는 먼저 저장할 수 있습니다.
- 월 마감 전에는 적용 인원이 0명이고 상태는 `월 마감 대기`로 표시됩니다.

### 3. 월 마감 기록 오류 수정

- 직원별 월 마감 원본까지 정상 저장된 기록만 월 마감 기록에 표시합니다.
- 과거 v1 테스트 행이나 불완전한 행은 목록에 표시하지 않습니다.
- 월 마감 기록에서 `마감 삭제`가 가능하며 삭제 후 이후 월 누적을 다시 계산합니다.

### 4. 월 마감 파일 보관함 추가

- 새 메뉴: `월 마감 파일 보관함`
- 경로·대상 월·파일 구분별로 계획표, 실제 근태표, 결과·수정본, 기타 엑셀을 직접 저장할 수 있습니다.
- 저장 파일을 다시 다운로드하여 수정한 뒤 재업로드할 수 있습니다.
- 저장된 계획표나 실제 근태표는 `분석에 불러오기` 버튼으로 바로 분석 화면에 넣을 수 있습니다.
- 월 마감 저장 시 사용한 계획표와 실제 근태표는 자동 보관되며, 같은 경로·월을 교체 저장하면 자동 보관 원본도 새 파일로 교체됩니다.
- 기존 D1에 파일을 여러 조각으로 나눠 파일당 최대 20MB까지 저장합니다. 별도 R2 설정은 필요하지 않습니다.
- R2 바인딩 이름을 `FILES`로 추가하면 파일은 자동으로 R2에 저장됩니다.

## 확정 판정 기준

### 출근기록이 필요한 계획

`근무`, `근무A`, `근무B`, `근무C`, `교육`, `오전반차`, `오후반차`, `공백`

실제 출근시간과 변경 출근시간이 모두 없으면 `근무인데 출근기록 없음`으로 표시합니다.

### 출근기록이 있으면 이상으로 보는 계획

`휴무`, `무급휴가`, `연차`, `공가`, `휴가`, `경조`, `대체휴일(1일)`, `대체휴일(0.5일)`, `보상휴가(1일)`, `보상휴가(0.5일)`

위 계획인데 출근기록이 있으면 `휴무·휴가인데 출근기록 있음`으로 표시합니다.

`기타`는 자동 단정하지 않고 계획·실제 근태 불일치 검토 대상으로 표시합니다.

## 배포 방법

### 1. 압축 해제

ZIP을 압축 해제한 뒤, 바깥 폴더 자체가 아니라 그 안의 아래 항목을 복사합니다.

- `public`
- `functions`
- `migrations`
- `README.md`

### 2. 기존 GitHub 저장소 폴더에 덮어쓰기

GitHub Desktop에서 `Repository → Show in Explorer`를 누르고 기존 `attendance-system` 폴더를 연 뒤 복사한 항목을 붙여넣습니다.

정상 구조:

```text
attendance-system
├─ public
├─ functions
├─ migrations
└─ README.md
```

### 3. Commit과 Push

GitHub Desktop 왼쪽 아래 Summary에 다음처럼 입력합니다.

```text
v4 사번 인식 및 월 마감 파일 보관함 수정
```

1. `Commit to main`
2. 상단 `Push origin`
3. Cloudflare Pages 배포가 `Success`가 될 때까지 기다립니다.

## D1 Console 실행 여부

이번 v4는 API가 필요한 테이블과 컬럼을 자동 점검하므로 `0002` 또는 `0003` SQL을 다시 수동 실행할 필요가 없습니다.

`migrations/0003_file_archive.sql`은 신규 DB를 직접 구성하거나 구조를 확인할 때만 사용하는 참고용 파일입니다.

## 배포 후 확인 순서

1. 운영 주소에서 관리자 로그인
2. 계획표와 실제 근태표 업로드 후 사번 인식 확인
3. `경로별 공통 대체휴무`에서 테스트 설정 1건 저장
4. 월 마감이 없다면 상태가 `월 마감 대기`, 적용 인원 0명인지 확인
5. `월 마감 기록`이 비어 있는지 확인
6. `월 마감 파일 보관함`에서 수정본 파일 직접 저장·다운로드 확인
7. 분석 후 월 마감 저장
8. 계획표와 실제 근태표가 보관함에 자동 생성되는지 확인
9. 같은 경로·같은 월을 다시 저장했을 때 월 마감과 자동 보관 원본이 교체되는지 확인
