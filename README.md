## v22 핵심 수정 — 증빙 O 선택 반영·발생일 소급 확정·상이 시트 대시보드

### 1. 출근 증빙 O 처리

- `출근 미등록` 시트 K열의 `O`를 사번과 `근태수정발생일` 기준으로 읽습니다.
- 증빙 날짜는 `상담사근태`에서 `출근`으로 확정하고 셀 채우기를 `#A9D08E`로 표시합니다.
- 계획이 근무였던 단순 미입력은 `출근 미등록`, `전체 요약본`, `매니저별 이상 근태`, `계획&근태 상이 인원`에서 제외합니다.
- 계획이 휴무·연차 등이었는데 실제 출근한 건은 출근 미등록에서는 제외하되 계획 상이에는 유지하고 `처리 완료`로 표시합니다.
- 실제 최종본의 머리글 `근태수정발생일`을 직접 인식하며, 열 위치가 달라져도 머리글 기준으로 찾습니다.

### 2. 지정 발생일 휴무·대체휴무 확정

- 계획 근무 계열은 출근 미입력이어도 대체휴무 부여를 유지합니다.
- 계획 `휴무`·미출근은 대체휴무를 부여하지 않고 발생 월의 기본 휴무 가능 수량만 부여 수량만큼 늘립니다.
- 계획 `휴무`·실제 출근은 기본 휴무 추가를 취소하고 대체휴무를 부여하며 계획 상이를 유지합니다.
- 발생일 이후 입사자는 계속 제외합니다.
- 미래 발생분도 사용 시작일이 대상 월 이전이면 해당 월 사용 가능 수량에 포함합니다.
- 발생일과 월 마감 기준일이 지난 뒤에는 일별 계획·근태 스냅샷으로 최종 판정하고, 미부여 대상의 lot는 사용 시작월까지 소급 재계산합니다.

### 3. 월 마감 재계산용 스냅샷

- 월 마감에 원 기본휴무 수량, 발생일 휴무 추가, 일별 계획·근태, 증빙 날짜를 함께 저장합니다.
- 부여 설정을 월 마감 뒤에 추가·수정해도 D1의 기존 마감 원본을 삭제하지 않고 누적 결과만 다시 계산합니다.
- 같은 경로·같은 월을 다시 저장하면 기존 closure ID를 유지한 채 해당 월의 하위 분석자료만 교체하는 기존 방식을 유지합니다.

### 4. `계획&근태 상이 인원` 디자인

- 네이비 보고용 대시보드, 상단 요약 카드 5개, 표 위 색상 안내를 적용합니다.
- 서울·경인·충청·경북·경남·전라별 그룹 행과 유형별 건수를 표시합니다.
- 13개 헤더와 1~7행을 고정하고, 구분·처리상태를 색상 배지로 표시합니다.
- 모든 시트의 눈금선을 숨깁니다.

> 기존 v21 이전 월 마감은 일별 스냅샷이 없으므로, 발생일 소급 확정을 정확히 적용하려면 해당 월을 계획표·근태 RAW·수정 최종본으로 한 번 다시 분석하여 같은 월 마감으로 교체 저장하세요.

## v21 핵심 수정 — 발생일은 근무계획 우선

발생일의 대체휴무·기본휴무 판정 순서를 다음으로 고정했습니다.

- 계획이 `근무·근무A·근무B·근무C·교육·반차·공백`이면 출근시간이 없어도 대체휴무를 우선 부여
- 계획이 `휴무`이고 실제 출근하지 않았으면 기본 휴무 가능 개수만 1일 추가
- 계획이 `휴무`여도 실제 출근했다면 기본 휴무 추가를 취소하고 대체휴무 1일 부여
- 계획 근무인데 출근기록이 없으면 대체휴무는 유지하고 출근 미등록·계획 상이는 별도 표시
- 사용 시작일이 발생일보다 빠른 미래 발생분은 기존처럼 선반영
- 월 마감에는 `occurrence_substitute_dates_json`을 저장하여 이후 누적 재계산에서도 계획 우선 판정을 유지

배포 후 기존 발생 월은 계획표와 근태표로 다시 분석하여 같은 월 마감을 교체 저장해야 새 판정이 과거 누적에도 반영됩니다.


## v20 핵심 수정

### 1. 발생일 근태에 따른 휴무·대체휴무 확정

- 발생일에 **실제 출근한 직원**: 대체휴무를 1인당 설정 수량만큼 확정 부여
- 발생일에 **계획이 휴무이고 출근기록이 없는 직원**: 대체휴무를 주지 않고 해당 월 기본 휴무 가능 수량을 같은 수량만큼 추가
- 계획이 휴무여도 실제 출근했다면 대체휴무는 부여하며 계획·근태 상이는 유지
- 발생일 이후 입사자는 자동 제외
- 사용 시작일이 발생일보다 빠르면 미래 발생분도 사용 가능 기간에 포함하여 선반영
- 발생일이 지난 뒤 실제 출근자는 대체휴무를 유지하고 휴무자는 잔여 대체휴무를 확정하지 않음

### 2. 출근 증빙 O 반영 오류 수정

- `출근 미등록` 시트의 K열 `O`·`○`·`ㅇ`·`Y`·`완료`를 인식
- `근태수정필요일`, `발생일`, `근무일자`, `일자`, `날짜` 머리글을 모두 인식
- 증빙 완료된 사번·일자는 실제근태가 원본에서 `미입력`이어도 최종적으로 `출근` 처리
- 상담사근태 셀을 출근 및 `#A9D08E`로 표시
- 동일 사번·일자를 출근 미등록, 계획&근태 상이, 전체 요약본, 매니저별 이상 근태에서 제거
- 월 마감 실제 출근일에도 반영하여 발생일 대체휴무 판정에 사용

### 3. 부여·연차 기록이 빈 화면으로 보이던 오류 방지

- 부여 현황 조회 시 누적 재계산을 강제로 실행하던 부작용 제거
- 부여 기록 저장은 성공했는데 재계산만 실패한 경우에도 저장 성공으로 유지하고 경고만 표시
- 누적 계산이 모두 끝난 뒤 기존 계산 결과를 교체하여 계산 도중 기존 잔여가 사라지지 않도록 수정
- 월 마감 저장과 누적 재계산 오류를 분리하여 재계산 오류 때문에 정상 저장된 월 마감 원본을 삭제하지 않음
- 연차·부여 조회 오류를 로그인 문구로 숨기지 않고 실제 오류 내용을 화면에 표시

### 4. 월 마감 비교 유지

- 기준 마감본 A와 비교 마감본 B를 별도 공간에서 사번·날짜 기준으로 비교
- 파일 보관함의 결과 파일을 계획표 입력란으로 보내지 않고 증빙 반영·기준본 A·비교본 B로 구분


기존 Cloudflare Pages·D1 구조를 유지하면서, 근태 월 마감과 별도로 **연차 최초 기준대장 + 월별 승인·반려 파일**을 누적 관리하는 버전입니다.

### 2. 월별 승인·반려 파일

매월 초 받은 `연차신청현황` 파일을 경로와 월별로 등록합니다.

- 승인된 연차·반차만 잔여 연차에서 차감
- 반려·취소·미사용·대기 건은 이력에는 남기되 잔여에서 차감하지 않음
- 같은 경로·같은 월 파일을 다시 등록하면 기존 월 자료를 완전히 교체
- 수정본을 재등록할 경우 마지막 파일이 최종 기준
- 근태 월 마감 시 선택한 연차신청현황도 해당 월 원장에 자동 저장
- 엑셀 내부 범위가 `A1`로 잘못 저장된 파일도 실제 셀 범위를 다시 찾아 전체 행을 읽음
- 선택한 월에 자료가 없으면 휴가일자가 가장 많은 실제 대상 월로 자동 보정

## v13 수정 사항

- `상담사근태` 요약 열을 **해당 월의 마지막 날짜 바로 다음 열**부터 배치합니다.
  - 2026년 6월 기준: `AS5`부터 시작
  - 31일인 달은 일자 열이 하나 더 있으므로 `AT5`부터 시작

- `AY 대체+보상 초과`가 실제로 0보다 큰 경우에만 빨간색으로 표시합니다.
- 6월에 대체휴무나 보상휴가를 사용하면 사용분이 누적 차감되어 7월의 `휴무 가능 개수` 괄호 안 숫자에 반영됩니다.

## v18 수정 사항 (이전 방식 기록)

> 아래 v18의 "휴무일 대체휴무 자동 사용" 방식은 v20에서 폐기되었습니다. 현재 적용 기준은 이 문서 맨 위의 v20 발생일 분기 규칙입니다.

### 1. 발생일 기준 대체휴무 자동 부여·자동 사용

- `대체휴무·보상휴가 부여` 화면에서 발생일을 여러 개 입력할 수 있습니다.
  - 줄바꿈, 쉼표, 공백으로 구분
  - 입력한 날짜마다 별도 부여 기록 생성
- 대상자는 각 발생일 기준으로 판단합니다.
  - `입사일 ≤ 발생일`: 부여 대상
  - `입사일 > 발생일`: 제외
  - 예: 2026-06-03 발생분은 2026-06-05 입사자를 제외
- 대체휴무 발생일에 다음 조건을 모두 충족하면 자동 사용으로 처리합니다.
  - 근무계획이 `휴무`
  - 실제 출근시간과 변경 출근시간이 모두 없음
  - 해당 직원에게 그 날짜의 대체휴무 부여분이 있음
- 자동 사용 시:
  - 상담사근태 일별 값은 `휴무` 유지
  - 기본 휴무 등록 개수에 1일 포함
  - 대체휴무 잔여에서 1일 차감
  - 실제 근태 열에 다른 문구가 있어도 출근시간이 없으면 지정일 휴무를 우선
- 근무계획은 `휴무`지만 출근시간이 있으면:
  - 대체휴무를 사용하지 않음
  - 잔여 1일 유지 또는 기존 차감 복원
  - `계획·실제 불일치`에는 계속 표시
- 계획에 이미 `대체휴일(1일/0.5일)`이 명시된 경우 자동 사용을 중복 추가하지 않습니다.
- 부여 기록을 삭제하거나 발생일을 변경하면 기존 월 마감 재계산 시 더 이상 유효하지 않은 자동 차감은 제거됩니다. 새로 추가한 발생일의 자동 사용은 해당 월을 다시 분석하고 월 마감 교체 저장해야 반영됩니다.

### 2. 출근 증빙 O 자동 반영

1. 기존 최종본의 `출근 미등록` 시트에서 `증빙여부(O 입력)` 열에 `O`를 입력합니다.
2. 근태 종합 점검의 `증빙 반영·비교 최종본`에 그 파일을 선택합니다.
3. 계획표·실제 근태표와 함께 다시 분석합니다.

반영 결과:

- 상담사근태 해당 일자를 `출근`으로 변경
- 셀 채우기 `#A9D08E`, 검은색 굵은 글씨
- `출근 미등록`에서 제외
- `계획&근태 상이 인원`의 `근무인데 출근기록 없음`에서 제외
- `전체 요약본` 미해결 건수에서 제외
- 웹의 매니저별 이상 근태와 엑셀 `매니저별 이상 근태`에서 제외
- 월 마감의 실제 출근일에도 저장되어, 이후 `특정일 실제 출근자만 부여` 판정에도 출근으로 인정

단, 계획이 휴무인데 실제 출근한 것처럼 **출근 증빙 후에도 실제 계획 불일치가 남는 경우**는 계획 상이에서 유지합니다.

### 3. 매니저별 상담사 이상 근태

- 웹 결과에 `매니저 수정요청` 탭과 매니저 선택 필터를 제공합니다.
- 근태 미입력, 휴무·휴가 출근, 계획 불일치, 휴무 초과, 대체·보상 초과, 연차 신청 불일치를 직원별로 묶습니다.
- 최종 엑셀에 `매니저별 이상 근태` 시트를 생성합니다.
- 증빙 완료된 일반 출근 미등록 건은 재분석 시 자동 제외됩니다.

### 4. 인력 매칭 메뉴명 변경

- 상단 `시스템 수정` 메뉴를 `인력 매칭`으로 변경했습니다.
- 기능과 월별 전체교체 정책은 그대로 유지합니다.

### 5. 월 마감 파일 보관함 교체 정책

- 수동 업로드도 같은 `경로 + 대상 월 + 파일 구분`이면 기존 파일을 삭제하고 마지막 업로드 파일로 교체합니다.
- 예: 전자랜드 2026-06 `결과·수정본`을 다시 올리면 이전 결과·수정본은 삭제되고 새 파일 1개만 남습니다.
- 다른 월, 다른 경로, 다른 파일 구분의 자료는 유지됩니다.

## v18 적용 후 권장 확인 순서

1. `인력 매칭`에 해당 월 인력자료가 등록되어 있는지 확인
2. 대체휴무 부여 화면에서 발생일과 입사일 기준 적용 인원을 확인
3. 계획 휴무·출근시간 없음 상담사가 발생일 자동 사용으로 잡히는지 확인
4. 계획 휴무·출근시간 있음 상담사가 계획 상이에 남고 대체휴무는 차감되지 않는지 확인
5. 출근 미등록 시트에 O 입력한 최종본을 증빙 파일로 재업로드해 오류 목록에서 빠지는지 확인
6. 최종본에 `매니저별 이상 근태` 포함 총 10개 시트가 생성되는지 확인
7. 같은 경로·월·파일 구분의 수정본을 보관함에 다시 올렸을 때 기존 파일이 교체되는지 확인
