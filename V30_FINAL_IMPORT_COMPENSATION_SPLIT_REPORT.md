# v30 이전 최종본 사용내역 가져오기 · 보상휴가 출근/미출근 분기

## 1. 이전 최종본 가져오기

관리자 화면의 **이전 최종본 휴가 사용내역 저장**에서 경로·대상 월·최종본 XLSX를 선택하면 다음 자료만 읽습니다.

- `상담사근태`가 포함된 시트 1개
- 사번·성명·매장·입사일 등 해당 시트의 인적사항
- 날짜별 셀에 화면상 표시된 최종값
- `대체휴무`, `대체휴일`, `보상휴가 1일`, `보상휴가 0.5일` 등의 사용표시
- 날짜별 출근시간 또는 `출근/근무/정상` 표시

다음 자료는 사용하지 않습니다.

- 요약 수식과 휴무 개수 수식
- 다른 시트의 값
- 근무계획, 근태RAW, 연차RAW, 증빙 시트의 값
- 셀의 수식 문자열 자체

같은 경로·같은 월을 다시 저장하면 전용 최종본 가져오기 자료를 교체합니다. 정상 월 마감이 이미 존재하면 오류·연차·원본자료는 유지하고, 파일에 포함된 사번의 대체휴무·보상휴가 사용내역과 일별 최종표시만 갱신합니다.

상담사근태 시트의 지역장·매니저·지역·매장코드·매장명·포탈사번·입사일도 월별 인력 스냅샷에 보완 저장합니다. 사용자가 별도로 등록한 인력자료는 삭제하지 않습니다.

전자랜드 파일의 다른 시트에 값이 있어도 사용자가 요청한 기준에 따라 읽지 않습니다.

## 3. 보상휴가 발생일 기준

경로 일괄 보상휴가를 `발생일 2026-05-01`, `1인당 1.5일`로 등록한 경우:

### 홈플러스

- 5월 1일 실제 출근자: 기본휴무 6일 + 보상휴가 1.5일
- 5월 1일 미출근자: 기본휴무 7일 + 보상휴가 0일

### 전자랜드

- 2026년 5월 토·일 기본휴무: 10일
- 5월 1일 실제 출근자: 기본휴무 10일 + 보상휴가 1.5일
- 5월 1일 미출근자: 기본휴무 11일 + 보상휴가 0일

발생일 당일까지 입사한 사람만 대상이며, 발생일 이후 입사자는 제외합니다. 실제 출근 여부는 상담사근태의 일별 최종표시와 저장된 출근일을 기준으로 확정합니다.

## 4. 대체휴무 기준

대체휴무는 v29 기준을 그대로 유지합니다.

- 발생일의 출근·휴무 여부와 관계없이 동일 부여
- 기본휴무 수량을 가감하지 않음
- 발생일 당일까지 입사한 사람만 대상
- 기존 D1 부여·사용·잔여 누적 방식 유지

## 5. 데이터 보호

- D1 테이블 초기화 또는 전체 삭제 없음
- 사용자가 등록한 기존 인력자료 삭제 없음
- 같은 경로·같은 월 최종본 가져오기만 교체
- 정상 월 마감의 오류·연차·원본자료 유지
- 기존 대체휴무 계산 유지
