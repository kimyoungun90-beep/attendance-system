# v25 수정 보고서

## 1. Excel 복구 경고 원인

오류 메시지의 대상은 `/xl/worksheets/sheet3.xml`이며, 기준 양식에서 sheet3은 `출근 미등록` 시트입니다.

v24는 K열 O 입력 시 K열·처리상태·상담사근태 색상을 즉시 바꾸기 위해 XLSX 생성 후 `sheet3.xml`에 조건부서식 XML 문자열을 직접 삽입했습니다. 이 후처리는 Excel 버전과 기존 시트 XML 구성에 따라 요소 순서 또는 구조가 거부될 수 있었고, Excel이 해당 시트를 복구하면서 `출근 미등록` 내용이 빈 시트처럼 보이는 문제가 발생했습니다.

## 2. 파일 손상 방지 수정

- 문자열 치환 방식 제거
- `DOMParser`와 `createElementNS`로 정식 SpreadsheetML 노드 생성
- 조건부서식을 `conditionalFormatting` 허용 위치에 삽입
- 기존 동일 범위 규칙 제거 후 재생성하여 중복 방지
- `styles.xml`의 `dxfs` 실제 자식 수를 기준으로 dxfId 생성
- 저장 전 XML 파싱 검증
- ZIP 재생성 후 XML 재검증
- 검증 실패 시 조건부서식 후처리 전의 정상 XLSX를 반환하도록 안전장치 추가

K열 O 입력에 따른 다음 기능은 유지합니다.

- K열 증빙 입력칸 연녹색
- M열 `처리 완료` 연녹색
- 상담사근태 해당 날짜 `출근`
- 상담사근태 해당 셀 `#A9D08E`

## 3. 계획 공백 + 실제 출근 처리

다음 규칙으로 변경했습니다.

- 계획 공백 + 출근기록 있음: 정상 `출근`
- 계획 공백 + 변경 출근시간 있음: 정상 `출근`
- 계획 공백 + 출근기록 없음: `출ㆍ계 미입력`
- 근무계획 있음 + 출근기록 없음: `출근 미입력`

계획이 공백이어도 실제 출근이 확인된 날짜는 아래 목록에서 제외됩니다.

- 출근 미등록
- 계획&근태 상이 인원
- 전체 요약본
- 매니저별 이상 근태

계획과 출근이 모두 없던 날짜도 K열 O 증빙으로 출근이 확인되면 `계획 미입력` 검토를 다시 만들지 않고 정상 출근으로 확정합니다.

## 4. 증빙 파일 재업로드

`출근 미등록` 시트 K열에 O를 입력하고 저장한 최종본을 증빙 반영에 다시 넣으면 G열 사번 + H열 발생일 + K열 O를 기준으로 읽습니다. 머리글이 손상되거나 지워진 경우에도 G/H/K 고정 위치를 보조 기준으로 사용합니다.
