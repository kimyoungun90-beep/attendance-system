# v26 수정 보고서

## 1. 계획 미입력 + 실제 출근

- 근무계획이 공백이어도 출근시간, 변경 출근시간 또는 실제근태의 `출근/근무/정상` 값이 있으면 실제 출근으로 인정합니다.
- `상담사근태` 해당 날짜는 `출근`으로 표시됩니다.
- `출근 등록 횟수`와 `총 등록 현황`에 포함됩니다.
- `출근 미등록`, `전체 요약본`, `매니저별 이상 근태`에는 넣지 않습니다.
- 근무계획 누락은 `계획&근태 상이 인원`의 `검토 필요`에만 남깁니다.
- 계획과 출근이 모두 없던 건에 K열 O를 입력한 경우에도 출근은 인정하되 `계획 미입력` 검토로 전환합니다.

## 2. 총 등록 현황 30일 오집계

기존 Excel 수식은 정확히 `미입력`인 셀만 제외하여 `출근 미입력`, `계획 미입력`, `출ㆍ계 미입력`을 등록 일수로 세고 있었습니다.

수식을 다음 기준으로 변경했습니다.

- 빈 셀 제외
- `미입력` 문구가 포함된 모든 셀 제외
- 정상 출근 및 정상 휴무·휴가 코드는 포함

따라서 계획과 출근이 모두 없는 날짜는 총 등록 현황에서 빠지고, 계획이 없어도 실제 출근이 확인된 날짜는 `출근`으로 포함됩니다.

## 3. 출근 미등록 검은 채우기

조건부서식의 differential format을 `fgColor + solid` 방식으로 생성하면서 일부 Excel 다크 모드에서 검은 채우기로 렌더링되는 문제가 있었습니다. 기존 최종본 양식과 동일한 `bgColor` DXF 방식으로 변경했습니다.

- K열 O: 연한 민트색
- 처리 완료: 연녹색
- 미처리: 연분홍색
- 상담사근태 증빙 출근: `#A9D08E` 유지

## 4. 유지 사항

- 기존 D1 데이터 및 마이그레이션 유지
- 같은 경로·같은 월 재등록 시 기존 월 자료 교체 유지
- 출근증빙 G열 사번 + H열 발생일 + K열 O 처리 유지
- 발생일·대체휴무 계산 로직 유지
