# v29 발생일 동일 부여 기준 수정 보고서

## 요청 반영

기존에는 지정 발생일의 근무계획과 실제 출근 여부에 따라 다음처럼 나뉘었습니다.

- 계획 근무 또는 실제 출근: 대체휴무 부여
- 계획 휴무·미출근: 기본 휴무 수량 추가, 대체휴무 미부여
- 계획 휴무·출근: 기본 휴무 추가 취소, 대체휴무 부여

이 판정을 v29에서 제거했습니다.

## v29 최종 기준

1. 기본 휴무 수량은 전 직원에게 동일합니다.
   - 홈플러스: 월 6일
   - 전자랜드: 대상 월 토요일+일요일 개수
2. 발생일에 휴무였다는 이유로 기본 휴무를 추가하지 않습니다.
3. 경로 일괄 대체휴무는 발생일에 출근했는지, 휴무였는지와 관계없이 동일 부여합니다.
4. 발생일 당일 입사자는 포함하고 발생일 이후 입사자는 제외합니다.
5. 퇴사일, 제외 사번, 사번별 부여, 보상휴가의 기존 조건은 유지합니다.
6. 사용 시작일이 발생 월보다 빠른 선사용 설정도 유지합니다.
7. 같은 경로·같은 월 재등록 시 기존 월 자료를 교체하는 방식과 D1 데이터 구조는 변경하지 않았습니다.

## 코드 변경 위치

- `public/assets/app.js`
  - 발생일 계획·근태 분기 제거
  - 발생일 대상자는 항상 대체휴무 대상일로 저장
  - 발생일 기본휴무 가산을 0으로 고정
  - 현재 월 미리보기에서도 발생일 후 부여분 회수 제거
  - 관리자 부여 내역 설명 문구 변경
- `functions/_lib/recalculate.js`
  - 과거 월 재계산 시 발생일 계획·근태로 lot를 회수하지 않음
  - 기존 `occurrence_rest_days`를 기본 휴무 기준에 더하지 않음
- `functions/api/substitute-balances.js`
  - 과거 발생분을 실제 근태로 다시 필터링하던 로직 제거
- `functions/api/substitute-grants.js`
  - 부여 인원·총 부여량 집계에서 발생일 근태 필터 제거
- `functions/_lib/leave-eligibility.js`
  - 호환용 발생일 판정을 `부여 유지 / 기본휴무 가산 없음`으로 통일

## 유지한 기능

- v28의 보고 시트 디자인, 7행 고정, 전체 시트 70% 보기
- 증빙 O 반영
- 계획 공백+출근 정상 인정
- 대체휴무·보상휴가 색상과 초과일 빨간색 표시
- 같은 경로·같은 월 교체 저장
- 기존 D1 데이터 및 마이그레이션
