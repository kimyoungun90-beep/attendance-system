# v39 대체·보상휴가 발생일/사용기간 기준 재계산 수정

## 잘못된 원인

v38은 이전 최종본 상담사근태 우측 요약부의 `대체휴무 이월개수`를 확정 잔여처럼 저장했습니다.
이 값은 당시 수기 계산값이거나 일부 발생분만 반영된 값일 수 있어, 관리 화면에 별도로 등록한 5월·6월·7월 발생 대체휴무와 충돌했습니다.

김형삼(E247645)의 경우 이전 파일 수기값 2일이 우선 반영되어, 관리 화면의 발생일·사용기간으로 다시 계산한 값이 제대로 보이지 않았습니다.

## v39 기준

1. 이전 최종본에서는 `상담사근태` 날짜 셀의 실제 사용내역만 읽습니다.
   - 대체휴무 사용일
   - 보상휴가 사용일
2. 우측 요약부의 수기 `이월개수`는 잔여 계산에 사용하지 않습니다.
3. 잔여는 아래 자료로만 다시 계산합니다.
   - 부여 구분
   - 실제 발생 월/발생일
   - 사용 시작일
   - 사용 종료일
   - 사번별 월 사용내역
4. 발생 월과 사용 시작일은 서로 다른 역할입니다.
   - 대상자 판정: 실제 발생일 기준
   - 해당 월 사용 가능 여부: 사용 시작일~종료일 기준
5. 대체휴무는 발생일 당시 출근/휴무와 무관하게 대상자에게 동일 부여합니다.
6. 보상휴가는 발생일 실제 출근자에게 1.5일 부여하고, 미출근자는 보상휴가 대신 해당 월 기본휴무 +1을 적용합니다.

## 김형삼 검증 예시

현재 설정처럼 대체휴무 4건 모두 사용 시작일이 2026-05-01이고 5월에 1일 사용했다면:
- 총 대체휴무 4일
- 5월 사용 1일
- 6월 시작 이월 대체휴무 3일

발생일과 사용기간을 실제 월로 변경하면 설정대로 계산됩니다.
예: 5월 2건은 5월부터, 6월 1건은 6월부터, 7월 1건은 7월부터 사용 가능.

## D1

- 새 테이블/컬럼 추가 없음
- 기존 수기 이월값 컬럼은 남겨두지만 계산에서 사용하지 않음
- 배포 후 부여 목록 조회 또는 월 분석 시 자동 재계산
